サーバ詳細

サーバ詳細の表示

次の情報を表示および編集できます。

パラメータ

説明

名前(表示のみ)

サーバに割り当てられている名前。

タイプ

サーバタイプ(マスタまたはスレーブ)。このサーバをマスタサーバとして指定するには、［マスタとして設定］をクリックします。

DNS名

iFolderエンタープライズサーバのDNS名。

例: 192.168.1.1 or svr1.domain.com

Public URL(パブリックURL)

iFolderサーバに対応するパブリックIPアドレス。IPアドレスを変更するには、アドレスを編集し、［保存］をクリックして変更内容を保存します。

Private URL(プライベートURL)

iFolderサーバに対応するプライベートIPアドレス。このURLを使用して、iFolderドメイン内のサーバ間で通信できます。プライベートURLとパブリックURLは同じにすることができます。IPアドレスを変更するには、新しいアドレスを編集し、［保存］をクリックして変更内容を保存します。

マスタURL(スレーブサーバでのみ表示されます)

iFolderサーバに対応するIPアドレス。スレーブサーバは、このアドレスを使用してiFolderドメイン内のマスタサーバと通信します。

IPアドレスを変更するには、アドレスを編集し、［保存］をクリックして変更内容を保存します。


サーバステータスの表示

次の情報を表示できます。

パラメータ

説明

ステータス

サーバの現在のステータスが表示されます。

ユーザ

選択したサーバにプロビジョニングされているiFolderユーザの合計数が表示されます。

iFolder

選択したサーバに属しているiFolderの合計数が表示されます。


サーバレポートの表示

レポートを表示するには、ドロップダウンリストからレポートを選択します。

このオプションは、［レポート］ページの［Enable Reporting(レポーティングを有効にする)］オプションがオフになっている場合は無効です。

サーバログの表示

次のサーバログ情報を表示できます。

パラメータ

説明

システム

すべてのシステムアクティビティを追跡するsimias.logを表示するには、［システム］を選択します。

User Access(ユーザアクセス)

選択したサーバ上のユーザアクティビティを追跡するsimias.access.logを表示するには、［User Access(ユーザアクセス)］を選択します。

ログレベルの設定

次のログレベル情報を設定できます。

パラメータ

説明

すべて

Novellサポートが問題を解決する際に役立つすべてのサーバアクティビティを示します。

デバッグ

Novellサポートが問題をデバッグする際に役立つサーバアクティビティを示します。

Info

Novellサポートが問題を解決する際に役立つ基本的なサーバアクティビティを示します。デフォルトではこのオプションが選択されています。

Warn

発生する可能性があるすべてのシステムエラーを示します。

エラー

システム動作を停止するすべてのシステムエラーを示します。

Fatal

致命的なシステムエラーを示します。

オフ

ログを無効にします。


LDAP詳細

次のLDAP関連情報を編集できます。［保存］をクリックして、エントリを変更します。変更をキャンセルするには、［キャンセル］をクリックします。

<

パラメータ

説明

稼動期間

(表示のみ)

最初に同期が行われた日時が表示されます。

ステータス

LDAP同期エンジンの現在のステータスが表示されます。

Cycles(サイクル)

(表示のみ)

完了したID同期サイクルの合計数が表示されます。

Identity Sync(ID同期)

選択したiFolderドメイン内のiFolderユーザを、選択した間隔でLDAP情報に基づいて更新します。

［Identity Sync(ID同期)］フィールドに、時間間隔を分単位で指定します。iFolderユーザとLDAPユーザの同期を直ちに開始するには、［今すぐ同期］をクリックします。

Delete member grace interval(メンバー削除猶予間隔)

特定のユーザがLDAPから削除されてから、iFolderがそのユーザ情報をサーバから削除するまでの時間間隔を指定します。この機能を使用すると、ユーザが誤って削除された場合にデータを簡単に回復できます。


次のLDAP関連情報を編集できます。［編集］をクリックすると、新しいぺージが開き、エントリを編集できます。

パラメータ

説明

LDAPサーバ

LDAPサーバのアドレスを示します。

LDAP SSL

LDAP SSLが有効になっているかどうかを示します。LDAP SSL接続を有効にしたり無効にしたりすることもできます。

プロキシユーザ

プロキシユーザの識別名(DN)が表示されます。例: cn=SimiasProxy,o=novell

iFolderプロキシユーザは、指定したコンテナやグループ内のユーザのリスト、またはiFolder LDAP設定で定義されているユーザのリストを取得するために、LDAPサーバにアクセスする際に使用されるIDです。iFolderプロキシユーザは、iFolderのインストール中に作成されます。このIDには、LDAPディレクトリに対する読み込み権が必要です。

プロキシユーザパスワード

 

このパスワードは、iFolderがユーザをLDAPサーバと同期するときに、LDAPサーバでiFolderプロキシユーザを認証する場合に使用されます。

LDAPコンテキスト

［LDAPコンテキスト］には、すべてのLDAPコンテキストのリストが表示されます。iFolderは、リストのLDAPコンテキストからのみユーザを検索します。


データストア

データストアは、特定のサーバの複数のボリューム(マウントポイント)にまたがるiFolderストレージを表します。デフォルトでは、すべてのiFolderサーバに、無効にできないデフォルトのストアがあります。Webインタフェースを使用して、複数のデータストアを追加および設定し、iFolderデータをそれらのデータストアに分散できます。ユーザがiFolderをアップロードするときに、データストアの最大空き領域が確認され、iFolderデータは特定のデータストアに格納されます。その結果、負荷が分散されます。必要な数だけデータストアを追加できます。複数のデータストアがあれば、大規模展開のデータストレージ容量を、エンタープライズレベルの要件に合わせて拡張できます。

次のデータストア情報を表示できます。

パラメータ

説明

名前

データストアに指定した一意の名前が表示されます。

Full path(フルパス)

ボリュームがマウントされているデータストアのパスが表示されます。これは、Webインタフェースを使用してデータストアを追加するときに指定したデータパスです。

空き領域

ボリューム内の使用可能な領域を示します。

使用可能

データストアが有効になっているかどうかが表示されます。デフォルトのデータストアは無効にできません。

データストアの有効化または無効化

無効または有効にするデータストアを選択して、それぞれ［無効］または［有効］をクリックします。ユーザがiFolderをアップロードするときに、iFolderデータの格納に使用できる最大空き領域が確認されます。その間、無効になっているデータストアは常にスキップされます。
