iFolder詳細の管理

このインタフェースでは、選択したiFolderの所有権、ポリシー、およびメンバーシップを管理できます。

注意: セカンダリ管理者は、選択されたiFolderに対してセカンダリ管理者が持つ権利に基づいて、選択されたiFolderのポリシーを管理できます。

説明(編集可能)

iFolderの簡単な説明. 説明を編集して[保存]をクリックすると、変更内容を保存できます。

所有者

選択したiFolderの所有者のユーザ名。所有者は個々のユーザまたはLDAPGroupになります。iFolderの所有者がLDAPGroupである場合、そのLDAPGroupのすべてのメンバーに同じアクセス権が付与されます。孤立したiFolderの場合、iFolderが再割り当てされるか削除されるまでは、iFolder管理者ユーザが所有者になります。

所有者のユーザアカウント詳細を表示するには、その所有者のユーザ名のリンクをクリックします。

iFolderの所有者には、iFolderに対するフル制御アクセス権があります。所有者は、ユーザのメンバーシップとアクセス権を管理したり、メンバーのフル制御アクセス権を削除したりできます。

パス

サーバにおけるiFolderとそのデータの実際の場所。例:

/var/opt/novell/ifolder3/simias/SimiasFiles/e84fdc6e-3d51-49df-ae3f-8c9213c76994/<iFolder_Name>

この例で、e84fdc6e-3d51-49df-ae3f-8c9213c76994は、iFolder共有の一意のIDです。

変更日時

iFolderが最後に変更された日時.

ディレクトリ

iFolder内のファイルの合計数

Orphan (孤立)

［はい］の場合、iFolderが孤立していることを示します。

孤立したiFolderの場合、iFolderが新しい所有者に割り当てられるか、または削除されるまでは、iFolder管理者が所有者になります。

メンバー

メンバーの名前、タイプ、およびメンバーに割り当てられているアクセス権を表示できます。所有者の追加または削除、所有権の割り当て、および選択したメンバーへのアクセス権の設定が可能です。

メンバーの追加

  1. ［iFolder Detail (iFolder詳細)］ページで、［追加］をクリックします。

  2. メンバーとして追加するユーザまたはLDAPGroupを検索し、そのユーザ名の横のチェックボックスをオンにして、［追加］をクリックします。

    デフォルトでは、ユーザまたはLDAPGroupにはiFolderに対する読み込み専用アクセス権が与えられます。

  3. (オプション)ユーザのチェックボックスをオンにして、［権利］を選択します。次に、アクセス権として［Admin］、［読み書き］、または［読み込み専用］を指定し、［設定］をクリックします。

    ページが更新されるのを待ちます。ユーザのアイコンに新しいアクセス権が反映されます。

iFolderのアクセス権について

iFolderクライアントでは、iFolderメンバーに対して所有者および3つのレベルのアクセス権をサポートしています。

注意: LDAPGroupのメンバーは、そのLDAPGroupのアクセス権セットを継承します。

所有者
iFolderの所有者としての役割を果たす唯一のユーザまたはLDAPGroupです。通常は、そのiFolderを作成したユーザまたはLDAPGroupのメンバーになります。所有者がLDAPGroupである場合は、そのLDAPGroupのすべてのメンバーがそのiFolderの所有権を持つことになります。所有者であるユーザまたは所有者であるLDAPGroupのメンバーは、iFolderクライアントを使用して、そのiFolderの他のメンバーに所有権を譲渡できます。iFolder管理者ユーザは、所有権をiFolderサーバ上の任意のユーザまたはLDAPGroupに譲渡できます。

iFolderの所有者にはフル制御のアクセス権があります。 このユーザまたはLDAPGroupはiFolderに対する読み書き可能アクセス権を持ち、メンバーユーザおよびLDAPGroupのメンバーシップとアクセス権を管理したり、メンバーのフル制御アクセス権を削除したりできます。エンタープライズサーバでは、所有者のiFolderで使用されているディスク容量に応じて、エンタープライズサーバにおける所有者のユーザディスク割り当て量が少なくなります。

ユーザまたはLDAPGroupがiFolderシステムから削除された場合、そのユーザまたはLDAPGroupによって所有されていたiFolderは孤立します。孤立したiFolderは、一時的に所有者としてのiFolder管理者ユーザに割り当てられます。メンバーシップと同期は他のユーザおよびLDAPGroupに引き継がれますが、iFolder管理者ユーザは、孤立したiFolderを削除するか、または新しい所有者に割り当てるかを決定します。

フル制御
フル制御のアクセス権を持つ共有iFolderのメンバーです。フル制御のアクセス権を持つユーザまたはLDAPGroupは、iFolderに対する読み書き可能アクセス権を持ち、所有者以外のすべてのユーザのメンバーシップとアクセス権を管理します。

読み書き
共有iFolderのメンバーで、iFolder内のディレクトリとファイルに対する読書き可能アクセス権を持ちます。

読み込み専用
共有iFolderのメンバーで、iFolder内のディレクトリとファイルに対する読み込み専用アクセス権を持ちます。このメンバーは、iFolderファイルを別の場所にコピーしたり、iFolder以外の場所で変更したりできます。

メンバーのiFolderアクセス権の設定

  1. ［iFolder Details (iFolder詳細)］ページで、アクセス権を管理するメンバーの横のチェックボックスをオンにします。

  2. ［権利］ドロップダウンメニューから、［Admin］、［読み書き］、または［読み込み専用］を選択します。

    ページが更新されるのを待ちます。ユーザのアイコンに新しいアクセス権が反映されます。

メンバーの削除

  1. ［iFolder Details (iFolder詳細)］ページで、メンバーユーザの名前の横のチェックボックスをオンにします。
  2. ［削除］をクリックします。

ユーザのデータのローカルコピーはユーザのコンピュータに残りますが、ユーザはiFolderデータのサーバコピーにアクセスできなくなります。

所有者の変更

iFolderの所有者を変更すると、既存の所有者はiFolderのメンバーになり、読み書き可能アクセス権が割り当てられます。孤立したiFolderの場合、iFolderが新しい所有者に割り当てられるか、または削除されるまでは、iFolder管理者が所有者になります。新しい所有者が現在iFolderのメンバーでない場合は、参加登録通知がユーザのiFolderアカウントに送信されます。

  1. ［iFolder Details (iFolder詳細)］ページで、iFolderの新しい所有者として割り当てるユーザまたはLDAPGroupを検索します。

  2. ユーザ名の横のチェックボックスをオンにして、［所有者］をクリックします。

ポリシー

次の表に、特定のiFolderについて管理できるポリシーを示します。次の値を1つまたは複数変更し、［保存］をクリックして新しい設定を適用します。

Disable Synchronization (同期を無効にする)

このオプションを選択すると、通常は一時的に同期が無効になります。

iFolder内のデータを同期できるようにするには、このオプションの選択を解除します。

デフォルト値: 有効、はい

Disk Quota (ディスク割り当て量)

［制限容量］チェックボックスをオンにして、選択したiFolderの最大サイズ(MB)を指定します。

システム全体のiFolder割り当て量またはLDAPGroupレベルの割り当て量を有効にした場合に、ユーザ割り当て量がLDAPGroupの割り当て量またはシステムの割り当て量より少ないか多いときは、ユーザのアカウント割り当て量が優先されます。

デフォルト値: 無効、制限なし

(表示のみ)

File Size

 制限容量
選択したユーザアカウントについて、ユーザが所有するすべてのiFolderに対してiFolderユーザが使用できる最大合計ファイルサイズ(MB)を指定します。

Effective (有効)
ユーザに割り当てられている有効なファイルサイズ。

    重要: 割り当て超過の原因になるサイズのファイルは、正常に同期できません。そのようなファイルを同期しようとすると、ファイルの一部だけが同期されるため、データが破損します。

Excluded Files (除外ファイル)

選択したiFolderの同期の対象にするファイルタイプと同期の対象外とするファイルタイプのリストを指定します。

thumbs.dbおよび.DS_Storeという名前のファイルマネージャファイルは、同期されません。

除外フィルタにファイル拡張子を追加するには、*.mpgなどの拡張子を入力し、［追加］をクリックしてフィルタを適用します。

特定のファイルタイプを同期から除外するには、制限されているファイルタイプのリストでそのファイルタイプの横のチェックボックスをオンにして、［拒否］をクリックします。

特定のファイルタイプの同期を許可するには、制限されているファイルタイプのリストでそのファイルタイプの横のチェックボックスをオンにして、［許可］をクリックします。

制限されているファイルタイプのリストからファイルタイプを削除するには、そのファイルタイプの横のチェックボックスをオンにして、［削除］をクリックします。

デフォルト値: 無効、すべてのファイルタイプまたはシステム全体の設定を許可

Synchronization Interval (同期間隔)

［Synchronization Interval (同期間隔)］チェックボックスをオンにして、選択したiFolderの最小間隔設定を有効にします。次に、ユーザがiFolderクライアントに対して設定できる最小値を分単位で指定します。

この設定を無効にするには、［Synchronization Interval (同期間隔)］チェックボックスをオフにします。このオプションが無効になっている場合は、［制限なし］と表示されます。

このオプションが有効になっている場合は、最小同期間隔に従って、ユーザのクライアントがサーバ上のiFolderデータとローカルiFolderを確認できる最小間隔が分単位で指定されます。この確認により、ダウンロードまたはアップロードする必要があるファイルを特定できます。

有効な最小同期間隔は、常に次の設定の中で最大の値です。

  • ユーザポリシーまたはLDAPGroupポリシーが設定されていない限り、システムポリシー(デフォルトは5分)。LDAPGroupポリシーが設定されている場合、システムポリシーはLDAPGroupポリシーで上書きされます。ユーザポリシーが設定されており、そのユーザポリシーの値がシステムポリシーを上回るか下回る場合、LDAPGroupポリシーとシステムポリシーはユーザポリシーで上書きされます。

  • ローカルマシンのポリシー、またはサーバと同期を行うクライアントマシン上の設定

  • iFolder (コレクション)ポリシー

デフォルト値: 無効、iFolderクライアント設定の制限

共用

 オン
デフォルトでは、iFolderの共有は有効になっています。選択されたiFolderの共有を無効にする場合は［オン］を選択解除します。このポリシーを適用すると、選択されたiFolderを管理者またはiFolderの所有者が共有できなくなります。

取り消し
［取り消し］を選択すると、選択されたiFolderの共有メンバーのリストから、すべてのメンバーが削除されます。

    重要: システムレベルまたはユーザレベルで［共有］オプションを有効にしたり強制したりすると、これらのオプションは両方とも無効になります。
