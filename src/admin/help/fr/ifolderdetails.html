Gestion des détails iFolder

Cette interface permet de gérer la propriété, les stratégies et l'adhésion d'un dossier iFolder sélectionné.

Remarque : un administrateur secondaire peut gérer les stratégies d'un dossier iFolder sélectionné en se basant sur les droits qu'il détient sur ce dossier.

Nom du dossier iFolder

Affiche le nom assigné au dossier iFolder

Description (modifiable)

Affiche une brève description du dossier iFolder. Vous pouvez modifier la description et cliquer sur Enregistrer pour sauvegarder les changements.

Propriétaire

Affiche le nom d'utilisateur du propriétaire du dossier iFolder sélectionné. Le propriétaire peut être un utilisateur individuel ou un groupe LDAP. Si un groupe LDAP est le propriétaire du dossier iFolder, tous les membres du groupe LDAP disposent alors des mêmes droits d'accès. Pour les dossiers iFolder orphelins, l'utilisateur Admin iFolder est considéré comme propriétaire jusqu'à ce que le dossier iFolder puisse être réassigné ou supprimé.

Cliquez sur le lien du nom d'utilisateur du propriétaire pour afficher les détails de son compte utilisateur.

Le propriétaire du dossier iFolder possède le droit Contrôle total sur le dossier iFolder. Le propriétaire gère l'adhésion et les droits d'accès des utilisateurs et peut supprimer le droit Contrôle total pour n'importe quel membre.

Chemin d'accès

Affiche l'emplacement exact du dossier iFolder et de ses données sur le serveur. Par exemple :

/var/opt/novell/ifolder3/simias/SimiasFiles/e84fdc6e-3d51-49df-ae3f-8c9213c76994/<Nom_iFolder>

Dans cet exemple, e84fdc6e-3d51-49df-ae3f-8c9213c76994 désigne l'ID unique du partage iFolder.

Modifié le

Affiche l'heure et la date de la dernière modification du dossier iFolder.

Répertoires

Affiche le nombre total de répertoires contenus dans le dossier iFolder.

Fichiers

Affiche le nombre total de fichiers contenus dans le dossier iFolder.

Orphelin

Oui indique que le dossier iFolder est orphelin.

Pour les dossiers iFolder orphelins, l'Administrateur iFolder devient le propriétaire jusqu'à ce que le dossier iFolder puisse être réassigné à un nouveau propriétaire ou supprimé.

Membres

Vous pouvez affichez le nom des membres, leur type ainsi que les droits d'accès qui leur sont attribués. Vous êtes autorisé à ajouter ou supprimer un propriétaire, à assigner une propriété et à définir des droits d'accès pour un membre sélectionné.

Ajout d'un membre

  1. Sur la page Détails iFolder, cliquez sur Ajouter.

  2. Recherchez l'utilisateur ou le groupe LDAP à ajouter en tant que membre, cochez la case en regard du nom de l'utilisateur, puis cliquez sur Ajouter.

    Par défaut, l'utilisateur ou le groupe LDAP dispose d'un accès en lecture seule au dossier iFolder.

  3. (Facultatif) Cochez la case Utilisateur, puis sélectionnez Droits et spécifiez le droit d'accès en tant que Admin, Lecture/écriture ou Lecture seule, puis cliquez sur Définir.

    Attendez que la page soit rafraîchie. L'icône de l'utilisateur devrait refléter le nouveau droit d'accès.

Présentation des droits d'accès aux dossiers iFolder

Le client iFolder prend en charge le propriétaire et trois niveaux d'accès pour les membres d'un dossier iFolder :

REMARQUE : les membres d'un groupe LDAP héritent des droits d'accès définis pour ce groupe.

Propriétaire
Un dossier iFolder ne peut avoir qu'un seul propriétaire, à savoir un utilisateur ou un groupe LDAP. Il s'agit généralement de l'utilisateur ou du membre du groupe LDAP qui a créé le dossier iFolder. Si le propriétaire est un groupe LDAP, tous les membres de ce groupe sont propriétaires de ce dossier iFolder. L'utilisateur propriétaire ou les membres du groupe LDAP propriétaire peuvent utiliser le client iFolder pour transférer la propriété à un autre membre du dossier iFolder. L'utilisateur Admin iFolder peut transférer la propriété à n'importe quel utilisateur ou groupe LDAP sur le serveur iFolder.

Le propriétaire d'un iFolder dispose d'un droit de contrôle total sur le dossier. Cet utilisateur ou groupe LDAP dispose de droits de lecture et d'écriture sur le dossier iFolder, gère les membres et les droits d'accès des utilisateurs et groupes LDAP membres et peut supprimer le droit de contrôle total d'un membre. Avec un serveur d'entreprise, l'espace disque utilisé par les dossiers iFolder du propriétaire est déduit des quotas de disque utilisateur du propriétaire sur le serveur d'entreprise.

Si un utilisateur ou un groupe LDAP est supprimé du système iFolder, les dossiers iFolder détenus par cet utilisateur ou groupe sont orphelins. Les dossiers iFolder orphelins sont assignés temporairement à l'utilisateur Admin iFolder, qui est alors considéré comme propriétaire. L'adhésion et la synchronisation se poursuivent pour les autres utilisateurs et membres de groupes LDAP tandis que l'utilisateur Admin iFolder détermine si un dossier iFolder orphelin doit être supprimé ou assigné à un nouveau propriétaire.

Contrôle total
Membre du dossier iFolder partagé avec le droit de contrôle total. L'utilisateur ou le groupe LDAP possédant le droit Contrôle total dispose d'un accès lecture/écriture au dossier iFolder et gère l'adhésion et les droits d'accès de tous les utilisateurs, à l'exception du propriétaire.

Lecture/écriture
Membre du dossier iFolder partagé disposant du droit Lecture/Ecriture sur les répertoires et fichiers du dossier iFolder.

Lecture seule
Membre du dossier iFolder partagé disposant du droit Lecture seule sur les répertoires et fichiers du dossier iFolder. Ce membre peut copier un fichier iFolder à un autre endroit et le modifier en dehors du dossier iFolder.

Définition du droit d'accès au dossier iFolder pour un membre

  1. Sur la page Détails iFolder, cochez la case en regard du membre dont vous souhaitez gérer le droit d'accès.

  2. Sélectionnez le menu déroulant Droits, puis sélectionnez le droit Admin, Lecture/écriture ou Lecture seule.

    Attendez que la page soit rafraîchie. L'icône de l'utilisateur devrait refléter le nouveau droit d'accès.

Suppression d'un membre

  1. Sur la page Détails iFolder, cochez la case en regard du nom d'utilisateur du membre.
  2. Cliquez sur Supprimer.

La copie locale des données de l'utilisateur reste sur l'ordinateur de l'utilisateur, mais celui-ci n'a plus accès la copie serveur des données iFolder.

Changement de propriétaire

Lorsque vous modifiez le propriétaire d'un dossier iFolder, le propriétaire existant devient membre du dossier iFolder et se voit attribuer le droit Lecture/écriture. Pour les dossiers iFolder orphelins, l'Administrateur iFolder devient le propriétaire jusqu'à ce que le dossier iFolder soit réassigné à un nouveau propriétaire ou supprimé. Si le nouveau propriétaire n'est actuellement pas membre du dossier iFolder, une notification d'abonnement est envoyée sur le compte iFolder de l'utilisateur.

  1. Sur la page Détails iFolder, recherchez l'utilisateur ou le groupe LDAP que vous souhaitez assigner en tant que nouveau propriétaire du dossier iFolder.

  2. Cochez la case en regard du nom de l'utilisateur, puis cliquez sur Propriétaire.

Stratégies

Le tableau suivant liste les stratégies que vous pouvez gérer pour n'importe quel dossier iFolder. Modifiez une ou plusieurs des valeurs suivantes, puis cliquez sur Enregistrer pour appliquer les nouveaux paramètres :

Paramètre

Description

Désactiver la synchronisation

Sélectionnez cette option pour désactiver, généralement de manière temporaire, la synchronisation.

Désélectionnez cette option pour permettre la synchronisation des données au sein du dossier iFolder.

Valeur par défaut : Activé, Oui

Quota de disque

Cochez la case Limite, puis spécifiez la taille maximale (en Mo) pour le dossier iFolder sélectionné.

Si vous activez le quota iFolder à l'échelle du système ou au niveau du groupe LDAP, le quota d'un compte utilisateur prévaut s'il est inférieur ou supérieur au quota du groupe LDAP ou système.

Valeur par défaut : Désactivé, Aucune limite

Utilisé

(Affichage uniquement) 

Indique l'espace actuellement occupé par les données iFolder.

Disponible

(Affichage uniquement) 

Indique l'espace disponible sur le serveur pour les données iFolder.

Effectif

(Affichage uniquement) 

Indique l'espace réellement disponible sur le serveur pour les données iFolder.

Taille du fichier

 Limite
Spécifie la taille de fichier maximale totale (en Mo) qu'un utilisateur iFolder est autorisé à utiliser, parmi tous les dossiers iFolder qu'il possède pour le compte utilisateur sélectionné.

Effectif
Affiche la taille de fichier effective allouée à l'utilisateur.

    IMPORTANT : les utilisateurs ne peuvent synchroniser correctement les fichiers dont la taille entraînerait le dépassement d'un quota. S'ils tentaient toutefois cette opération, le fichier ne serait que partiellement synchronisé, ce qui provoquerait une corruption des données.

Fichiers exclus

Dresse une liste des types de fichiers à inclure ou exclure de la synchronisation pour le dossier iFolder sélectionné.

Les fichiers du gestionnaires de fichiers nommés thumbs.db et DS-Store ne sont jamais synchronisés.

Pour ajouter une extension de fichier au filtre d'exclusion, tapez l'extension (par ex. *.mpg), puis cliquez sur Ajouter pour appliquer le filtre.

Pour exclure un type de fichier des types de fichiers restreints, cochez la case en regard du nom du fichier, puis sélectionnez le type de fichier et cliquez sur Supprimer.

Valeur par défaut : Désactivé, Autoriser tous les types de fichiers ou les paramètres système.

Intervalle de synchronisation

Cochez la case Intervalle de synchronisation pour activer un intervalle minimal pour le dossier iFolder sélectionné, puis spécifiez la valeur minimale en minutes que les utilisateurs sont autorisés à définir sur leur client iFolder.

Pour désactiver ce paramètre, désélectionnez la case Intervalle de synchronisation. Si l'option est désactivée, la valeur indiquée est Illimité.

Si cette option est activée, l'intervalle de synchronisation minimal spécifie l'intervalle minimal en minutes au cours duquel le client de l'utilisateur peut consulter les données iFolder sur le serveur et les dossiers iFolder locaux afin d'identifier les fichiers qui doivent être téléchargés.

L'intervalle de synchronisation minimal effectif est toujours la plus grande valeur des paramètres suivants :

  • La stratégie système (valeur par défaut : 5 minutes), sauf si une stratégie utilisateur ou groupe LDAP est définie. Si une stratégie de groupe LDAP est définie, elle prévaut sur la stratégie système. Si une stratégie utilisateur est définie, celle-ci prévaut sur la stratégie de groupe LDAP ou système, peu importe que sa valeur soit supérieure ou inférieure.

  • La stratégie de la machine locale ou la configuration sur la machine du client synchronisée avec le serveur

  • La stratégie (de collection) du dossier iFolder

Valeur par défaut : Désactivé, Limite définie par le client iFolder

Partage

 Actif
Par défaut, le partage iFolder est activé. Déselectionnez Actif pour désactiver le partage du dossier iFolder sélectionné. Après l'application de cette stratégie, le dossier iFolder sélectionné ne peut pas être partagé ni par l'Admin ni par le propriétaire du dossier iFolder.

Révoquer
Sélectionnez Révoquer pour supprimer tous les membres de la liste de membres partagés pour le dossier iFolder sélectionné.

    IMPORTANT : ces deux options sont désactivées si vous activez et appliquez l'option Partage au niveau du système ou de l'utilisateur.
