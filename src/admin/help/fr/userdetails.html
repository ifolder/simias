Détails de l'utilisateur

Cette interface permet d'afficher les détails de l'utilisateur et de gérer les stratégies utilisateur.

REMARQUE : le terme utilisateurs iFolder fait référence à la fois aux utilisateurs individuels et aux groupes LDAP.

Détails de l'utilisateur

La page Détails de l'utilisateur affiche les détails de l'utilisateur suivants pour le compte iFolder de l'utilisateur sélectionné.

Paramètre

Description

Nom d'utilisateur

Affiche le nom d'utilisateur assigné au compte utilisateur, p. ex. jdupont ou jacques.dupont@exemple.com.

Nom complet

Affiche les nom et prénom du compte utilisateur.

Type de membre

Affiche le type membre de l'utilisateur logué. Dans le cas d'un utilisateur individuel, l'interface affiche également une option pour les groupes d'utilisateurs. Si l'utilisateur est membre d'un groupe LDAP, l'interface reprend tous les membres du groupe LDAP sous l'option Membres de groupe.

Contextes LDAP

Le contexte de l'arborescence LDAP est utilisé pour donner aux utilisateurs accès à iFolder.

Dernier login

Affiche l'heure à laquelle l'utilisateur s'est logué au système iFolder pour la dernière fois.

Quota de disque du groupe Représente le quota de disque total d'un groupe.

REMARQUE : l'administrateur secondaire ne peut pas modifier le quota de disque du groupe.

Espace disque utilisé Représente l'espace disque total utilisé par un groupe.

Groupes d'utilisateurs (applicable uniquement aux utilisateurs individuels)

Enumère tous les groupes auxquels appartient l'utilisateur sélectionné.

Membres de groupe (applicable uniquement aux groupes LDAP)

Liste tous les membres qui appartiennent au groupe LDAP sélectionné.

Dossiers iFolder

La page Détails de l'utilisateur affiche les dossiers iFolder que l'utilisateur possède ou partage. Cliquez sur l'onglet Tout pour lister tous les dossiers iFolder avec propriétaire et partagés. Pour afficher le dossier iFolder appartenant à l'utilisateur, cliquez sur l'onglet Avec propriétaire. L'onglet Partagé liste tous les dossiers iFolder partagés pour le compte utilisateur actuel.

Activation ou désactivation d'un dossier iFolder

  1. Localisez le dossier iFolder à gérer, puis cochez la case en regard de celui-ci.

  2. Cliquez sur Activer pour activer le dossier iFolder.

    L'utilisateur peut alors se loguer et synchroniser les dossiers iFolder.

  3. Cliquez sur Désactiver pour désactiver le dossier iFolder.

    Si l'utilisateur est logué pendant que vous effectuez cette modification, sa session reste active jusqu'à ce qu'il se délogue. La stratégie prendra effet la prochaine fois que l'utilisateur tentera de se loguer au compte. Pour que le verrouillage prenne effet immédiatement, vous devez redémarrer les services Apache pour le serveur iFolder. Toutes les sessions actives seront alors déconnectées, y compris celle de l'utilisateur

Suppression d'un dossier iFolder

  1. Localisez le dossier iFolder à supprimer; puis cochez la case en regard de celui-ci.

  2. Cliquez sur Supprimer.

Création d'un dossier iFolder

  1. Sur la page Détails de l'utilisateur, cliquez sur Nouveau.

  2. Pour créer un nouveau dossier iFolder, suivez les instructions indiquées sur la nouvelle page qui apparaît.

Stratégies

Cette interface permet de gérer les stratégies utilisateur pour un compte utilisateur spécifié.

REMARQUE : les stratégies définies pour un groupe LDAP sont applicables à ses membres.

Modification des stratégies utilisateur

Le tableau suivant liste les stratégies que vous pouvez gérer pour n'importe quel compte utilisateur iFolder. Modifiez une ou plusieurs des valeurs suivantes, puis cliquez sur Enregistrer pour appliquer les nouveaux paramètres :

Paramètre

Description

Compte

Indique si l'utilisateur est actuellement autorisé à se loguer pour synchroniser des dossiers iFolder.

Cochez la case pour ne pas autoriser le compte à se loguer.

Désélectionnez la case pour autoriser le compte à se loguer.

Si l'utilisateur est logué pendant que vous effectuez cette modification, sa session reste active jusqu'à ce qu'il se délogue. La stratégie prendra effet la prochaine fois que l'utilisateur tentera de se loguer au compte. Pour que le verrouillage prenne effet immédiatement, vous devez redémarrer les services Apache pour le serveur iFolder. Toutes les sessions actives seront alors déconnectées, y compris celle de l'utilisateur

Valeur par défaut : activé

Nombre de dossiers iFolder par utilisateur

Spécifie le nombre maximum de dossiers iFolder qu'un utilisateur peut posséder. Une fois cette stratégie appliquée, l'utilisateur ne peut détenir qu'un nombre limité de dossiers iFolder. L'utilisateur qui dépasse cette limite reçoit un message d'erreur indiquant la violation de stratégie. Si la limite est zéro, l'utilisateur ne peut créer aucun dossier iFolder.

Sélectionnez la case à cocher en regard de Limite pour activer la limite du nombre de dossiers iFolder par utilisateur et spécifiez ce nombre dans le champ.

La configuration de la stratégie n'affecte pas le nombre de dossiers iFolder que l'utilisateur possède déjà. Si le nombre de dossiers iFolders que possède l'utilisateur dépasse déjà la limite que vous fixez, il peut toujours posséder ces dossiers iFolder.

Une stratégie de niveau utilisateur prévaut sur une stratégie de niveau groupe LDAP ou système.

Valeur par défaut : désactivé, aucune valeur définie

Quota de disque

Spécifie l'espace total maximal que les données iFolder d'un utilisateur sont autorisées à utiliser, parmi tous les dossiers iFolder possédés par l'utilisateur. Le quota peut être imposé par la stratégie système. Les stratégies utilisateur remplacent les stratégies système.

Limite
Spécifie, pour le compte utilisateur sélectionné, l'espace total maximal (en Mo) que les données iFolder d'un utilisateur sont autorisées à utiliser, parmi tous les dossiers iFolder que l'utilisateur possède.

Désélectionnez cette option si aucun quota utilisateur n'est défini ou pour accepter le quota de niveau groupe LDAP ou le quota à l'échelle du système pour le compte utilisateur sélectionné.
Sélectionnez cette option pour appliquer un quota utilisateur, puis spécifiez le quota d'espace total (en Mo) pour le compte utilisateur sélectionné.

Utilisé
Spécifie l'espace total actuellement occupé sur le serveur par tous les dossiers iFolder appartenant à l'utilisateur sélectionné.

Disponible
Affiche l'espace libre disponible sur le serveur.

Effectif
Affiche l'espace effectif alloué sur le serveur

Si l'espace total occupé par les données iFolder avoisine un quota applicable (système, groupe LDAP, utilisateur ou iFolder), l'utilisateur devrait ne plus synchroniser les fichiers jusqu'à ce qu'une ou plusieurs des tâches suivantes créent suffisamment d'espace pour synchroniser sans risque les fichiers de l'utilisateur dans le dossier iFolder qui contient le fichier :

  • Le quota à l'échelle du système, au niveau du groupe LDAP ou utilisateur pour le propriétaire du dossier iFolder et le quota iFolder sont modifiés au besoin.

  • Des fichiers sont transférés d'un des dossiers iFolder appartenant à l'utilisateur vers un autre emplacement où ils n'affectent plus le quota effectif, ou alors supprimés afin de libérer de l'espace.

IMPORTANT :si vous activez une limite d'espace utilisateur inférieure à l'espace total actuellement employé par un utilisateur pour stocker ses données iFolder, les données de l'utilisateur ne sont plus synchronisées jusqu'à ce que la taille des données descende sous la limite ou que le quota soit augmenté pour atteindre une valeur supérieure à l'espace total occupé par les données de l'utilisateur.

Valeur par défaut : Désactivé et Aucun quota

Taille du fichier

 Limite
Spécifie la taille de fichier maximale totale (en Mo) qu'un utilisateur iFolder est autorisé à utiliser, parmi tous les dossiers iFolder qu'il possède pour le compte utilisateur sélectionné.

Effectif
Affiche la taille de fichier effective allouée à l'utilisateur.

    IMPORTANT : les utilisateurs ne peuvent synchroniser correctement les fichiers dont la taille entraînerait le dépassement d'un quota. S'ils tentaient toutefois cette opération, le fichier ne serait que partiellement synchronisé, ce qui provoquerait une corruption des données.

Valeur par défaut : Désactivé, Aucune limite

Fichiers exclus

Dresse une liste des types de fichiers à inclure ou exclure de la synchronisation pour le compte de l'utilisateur sélectionné.

Les fichiers du gestionnaires de fichiers nommés thumbs.db et DS-Store ne sont jamais synchronisés. Vous n'avez pas besoin de les conserver. De plus, leur synchronisation entraîne de nombreuses erreurs de conflit de fichier. Si vous n'avez pas défini de restrictions particulières pour cet utilisateur, ce champ liste les fichiers thumbs.db et DS_Store dans les restrictions de type de fichier appliquées à l'ensemble du système. Après avoir défini des restrictions de type de fichier spécifiques pour l'utilisateur, les paramètres de ce dernier seront alors affichés. Même si les restrictions des fichiers thumbs.ds et DS_Store ne sont pas affichées, elles sont toujours appliquées. Il est impossible de les remplacer.

Pour ajouter une extension de fichier à un filtre d'inclusion ou d'exclusion, tapez l'extension (p. ex. *.mpg), puis cliquez sur Ajouter pour appliquer le filtre.

Pour exclure un type de fichier de la synchronisation, cochez la case en regard du type de fichier dans la liste des types de fichiers restreinte, puis cliquez sur Refuser.

Pour autoriser la synchronisation d'un type de fichier, cochez la case en regard du type de fichier dans la liste des types de fichiers restreinte, puis cliquez sur Autoriser.

Pour supprimer un type de fichier de la liste des types de fichier restreinte, cochez la case en regard du type de fichier, puis cliquez sur Supprimer.

Valeur par défaut : .DS_Store et Thumbs.db

Synchronisation

Spécifie l'intervalle de synchronisation minimal en secondes. Vous pouvez définir un intervalle de synchronisation compris entre 5 et 999 inclus. Si aucune valeur n'est définie, la valeur est 5.

Désélectionnez la case pour ne définir aucun intervalle de synchronisation ou pour accepter la configuration système pour le compte utilisateur. Si aucune valeur n'est définie pour les stratégies système ou utilisateur, la valeur indiquée est Illimité.

Intervalle
Cochez la case pour activer un intervalle de synchronisation minimal, puis spécifiez l'intervalle minimal (en minutes). Par exemple, une valeur pratique est 10 minutes.

Effectif
Spécifie l'intervalle de synchronisation actuel. Par exemple, si l'utilisateur définit un intervalle de synchronisation inférieur à l'intervalle minimal déterminé par le système, la configuration système s'applique.

L'intervalle de synchronisation minimal effectif est toujours la plus grande valeur des paramètres suivants :

  • La stratégie système (valeur par défaut : 0), sauf si une stratégie de groupe LDAP ou utilisateur est définie. Si une stratégie utilisateur est définie, celle-ci prévaut sur la stratégie système ou groupe LDAP, et ce que sa valeur soit inférieure ou supérieure.

  • La stratégie de la machine locale ou la configuration sur la machine du client synchronisée avec le serveur.

  • La stratégie (de collection) du dossier iFolder.

Valeur par défaut : Désactivé ou Stratégie système

le codage

Spécifie la stratégie de codage pour le compte de l'utilisateur sélectionné. Le quota peut être imposé par la stratégie système (globale). La stratégie système remplace les stratégies utilisateur.

Actif
Sélectionnez Actif pour activer la fonction de codage pour l'utilisateur sélectionné. Cette fonction permet à l'utilisateur de définir la stratégie de codage pour ses dossiers iFolder.

Désélectionnez Actif pour désactiver la fonction de codage pour l'utilisateur sélectionné.

Appliqué
Sélectionnez Appliqué pour activer la fonction de codage pour l'utilisateur sélectionné. Lorsque cette fonction est définie sur Appliqué, l'utilisateur ne peut pas modifier les paramètres de codage pour ses dossiers iFolder.

IMPORTANT :Cette option est activée uniquement si la stratégie de codage à l'échelle du système est définie sur Actif.

Partage

 Actif
Par défaut, le partage iFolder est activé. Désélectionnez Actif et cliquez sur Enregistrer pour désactiver le partage pour l'utilisateur sélectionné. Après l'application de cette stratégie, l'utilisateur sélectionné ne peut plus partager ses dossiers iFolder avec d'autres. Vous pouvez toutefois toujours modifier la stratégie au niveau iFolder.

Appliquer
Sélectionnez Appliquer pour appliquer la stratégie définie pour l'utilisateur sélectionné. Après l'application de cette stratégie, l'utilisateur ne peut plus partager ses dossiers iFolder.

Révoquer
Sélectionnez Révoquer pour supprimer les membres partagés de tous les dossiers iFolder appartenant à l'utilisateur sélectionné.
