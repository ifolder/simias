Détails du serveur

Affichage des détails du serveur

Vous pouvez afficher et modifier les informations suivantes :

Paramètre

Description

Nom (affichage uniquement)

Affiche le nom assigné au serveur.

Type

Affiche le type de serveur (Maître ou Esclave). Cliquez sur Définir en tant que maître pour configurer ce serveur en tant que serveur maître.

Nom DNS

Affiche le nom DNS du serveur d'entreprise iFolder.

Par exemple : 192.168.1.1 ou svr1.domain.com.

URL publique

Affiche l'adresse IP publique correspondant au serveur iFolder. Pour changer l'adresse IP, modifiez l'adresse spécifiée et cliquez sur Enregistrer pour sauvegarder les modifications lorsque vous avez terminé.

URL privée

Affiche l'adresse IP privée correspondant au serveur iFolder. Celle-ci permet la communication entre les serveurs au sein du domaine iFolder. L'URL publique et l'URL privée peuvent être identiques. Pour changer l'adresse IP spécifiée, tapez la nouvelle adresse et cliquez sur Enregistrer pour sauvegarder les modifications lorsque vous avez terminé.

URL principale (affichée uniquement pour les serveurs esclaves)

Adresse IP correspondant au serveur iFolder. Grâce à cette adresse, le serveur esclave communique avec le serveur maître dans le domaine iFolder.

Pour changer l'adresse IP, modifiez l'adresse spécifiée et cliquez sur Enregistrer pour sauvegarder les modifications lorsque vous avez terminé.

Vous pouvez afficher les informations suivantes concernant les journaux du serveur :

Paramètre

Description

Système

Sélectionnez Système pour afficher le journal simias.log qui procède au suivi de toutes les activités du système.

Vous pouvez définir les informations suivantes concernant le niveau de consignation :

Paramètre

Description

Affiche les erreurs système fatales.

Synchronisation d'identités

Met à jour, à l'intervalle que vous choisissez, les utilisateurs iFolder et le domaine iFolder sélectionné à partir des informations LDAP.

Spécifiez l'intervalle en minutes dans le champ Synchronisation d'identités et cliquez sur Synchroniser maintenant pour commencer immédiatement la synchronisation des utilisateurs iFolder avec les utilisateurs LDAP.

Supprimer l'intervalle bonus du membre

Spécifie l'intervalle avant que iFolder ne supprime du serveur les informations de l'utilisateur, une fois que celui-ci est supprimé du LDAP. Cette fonction vous permet de récupérer les données si l'utilisateur est supprimé accidentellement.


Vous pouvez éditer les informations suivantes relatives au protocole LDAP. Cliquez sur Modifier pour ouvrir une nouvelle page dans laquelle vous pouvez modifier les entrées.

Paramètre

Description

Serveur LDAP

Affiche l'adresse du serveur LDAP.

SSL LDAP

Indique si SSL LDAP est activé ou non. Cette option vous permet également d'activer ou de désactiver la connexion SSL LDAP.

Utilisateur proxy

Indique le nom distinctif (DN) de l'utilisateur proxy. Par exemple, cn=SimiasProxy,o=novell

L'utilisateur proxy iFolder est l'identité utilisée afin d'accéder au serveur LDAP pour récupérer une liste des utilisateurs appartenant aux conteneurs ou groupes spécifiés ou des utilisateurs définis dans les paramètres LDAP iFolder. L'utilisateur proxy iFolder est créé lors de l'installation de iFolder. Cette identité doit posséder le droit de lecture sur l'annuaire LDAP.

Mot de passe de l'utilisateur proxy 

 

Le mot de passe sert à authentifier l'utilisateur proxy iFolder sur le serveur LDAP lorsque iFolder synchronise les utilisateurs avec le serveur LDAP.

Contexte LDAP

Liste tous les contextes LDAP. iFolder recherche les utilisateurs uniquement dans les contextes LDAP listés.


Banque de données

La zone de stockage de données désigne le stockage iFolder qui peut être réparti sur plusieurs volumes (points de montage) sur un serveur donné. Par défaut, chaque serveur iFolder dispose d'une zone de stockage par défaut qui ne peut pas être désactivée. Avec l'interface Web, vous pouvez ajouter et configurer plusieurs zones de stockage afin d'équilibrer la charge des données iFolder. Lorsqu'un utilisateur télécharge un dossier iFolder, il vérifie quelle zone de stockage dispose de l'espace libre maximal et enregistre les données iFolder dans cette zone de stockage, équilibrant ainsi la charge. Vous pouvez ajouter autant de zones de stockage de données que vous le souhaitez. L'existence de plusieurs zones de stockage de données permet donc d'optimiser la capacité de stockage de données dans le cadre d'un vaste déploiement pour ainsi répondre aux exigences de niveau entreprise.

Vous pouvez afficher les informations suivantes concernant les zones de stockage de données :

Paramètre

Description

Nom

Indique le nom unique que vous avez spécifié pour la zone de stockage de données.

Chemin d'accès complet

Indique le chemin d'accès à la zone de stockage de données, sur laquelle est monté le volume. Il s'agit du chemin d'accès aux données que vous avez spécifié lors de l'ajout de la zone de stockage avec l'interface Web.

Espace disponible

Indique l'espace disponible dans le volume.

Activé

Indique si la zone de stockage de données spécifiée est activée ou non. La zone de stockage de données par défaut ne peut pas être désactivée.

Activation ou désactivation d'une zone de stockage de données

Sélectionnez la zone de stockage de données que vous souhaitez activer ou désactiver et cliquez respectivement sur Activer ou Désactiver. Lorsque l'utilisateur télécharge un dossier iFolder, les zones de stockage de données désactivées sont toujours ignorées lorsque l'utilisateur recherche la zone de stockage disposant de l'espace libre maximal pour y enregistrer les données iFolder.
