Orden de muestra de nombre completo

Permite definir el orden en que aparecerá el nombre completo del usuario. Seleccione la opción (Nombre, Apellido) para ver el nombre seguido del apellido. Si desea ver el apellido seguido del nombre, seleccione la opción (Apellido, Nombre).

Para que los cambios surtan efecto, es necesario realizar una sincronización de LDAP programada o realizar usted mismo una manual. Para llevar a cabo una sincronización de LDAP manual:

  1. En la consola de Web admin, haga clic en la pestaña Servidores, seleccione el servidor y, a continuación, vaya a la página Información del servidor.

  2. En la sección Información de LDAP, haga clic en el botón Sincronizar ahora.

Gestionar cuota de grupo mediante

Permite definir cómo se gestionará la cuota de disco en conjunto para los grupos.

Seleccione la opción Consola de administrador para que los administradores puedan gestionar explícitamente la cuota de disco para grupos y miembros de un grupo. Cuando seleccione esta opción, se limitará la cuota de disco asignada a los usuarios para que no supere el uso de disco en conjunto del grupo. La opción Consola de administrador está seleccionada por defecto.

Seleccione la opción Motor de sincronización para permitir al motor de sincronización gestionar la cuota de disco en conjunto en grupos en el sistema secundario. El motor de sincronización garantiza que la cuota de disco para todos los usuarios de un grupo no superará la cuota de disco en conjunto del grupo.

 

Grupos segregados Permite segregar grupos en entidades independientes y garantizar que los miembros de otro grupo no pueden acceder a los miembros de un grupo para compartir carpetas iFolder. Seleccione el recuadro de verificación Crear grupos segregados para habilitar el uso compartido de carpetas iFolder sólo entre los miembros del mismo grupo.

Nota: un grupo LDAP no puede ser un administrador principal o secundario.

Administradores de iFolder

El usuario Admin de iFolder es el administrador principal del servidor empresarial iFolder. Cuando las carpetas iFolder se quedan huérfanas, la propiedad se transfiere al usuario Admin de iFolder para que éste la reasigne a otro usuario o bien suprima la carpeta. El usuario Admin de iFolder se especifica inicialmente durante la configuración del servidor empresarial iFolder en YaST.

Es necesario asignar el usuario Admin de iFolder para que el administrador de iFolder pueda realizar las tareas de gestión. iFolder realiza un seguimiento del usuario mediante el GUID del objeto LDAP, permitiéndole que pertenezca a cualquier contenedor o grupo LDAP del árbol, incluso a los que no se hayan identificado como DN de búsqueda. Es posible realizar un seguimiento de los movimientos del usuario en cualquier parte del árbol, ya que éste se conoce por el GUID y no por el DN de usuario.

El derecho de administrador de iFolder puede asignarse a otros usuarios para que éstos también puedan gestionar los servicios de iFolder para el servidor seleccionado. Sólo los usuarios que se encuentren en uno de los DN especificados en el DN de búsqueda de LDAP podrán utilizarse de manera equivalente al usuario Admin de iFolder.

Si asigna el derecho de administrador de iFolder a otros usuarios, éstos se regirán por la relación de la lista y el DN de búsqueda. Si suprime el usuario, elimina el DN del usuario de la lista, elimina los DN de búsqueda o mueve el usuario a un DN que no se encuentra entre los DN de búsqueda, éste se eliminará de la lista y quedará desprovisto del derecho de administrador de iFolder.

El usuario Admin de iFolder o el administrador principal crea el administrador secundario y le asigna grupos.

IMPORTANTE: no es posible asignar el derecho de usuario Admin a un grupo LDAP.

Administrador principal

La página Información de usuario muestra las carpetas iFolder en propiedad o compartidas por el usuario Admin. Directivas

En la tabla siguiente se enumeran las directivas del sistema que pueden gestionarse para un sistema de iFolder determinado. Haga clic en Guardar para aplicar las modificaciones:

Parámetro

Descripción

Carpetas iFolder por usuario

Especifica el número máximo de carpetas iFolder permitidas por usuario. Tras aplicar esta directiva, cada usuario está limitado a poseer un determinado número de carpetas iFolder. Los usuarios que superen este límite recibirán un mensaje de error sobre la infracción de la directiva. Si el límite es cero, los usuarios no pueden crear ninguna carpeta iFolder.

El valor de configuración de la directiva no afecta al número de carpetas iFolder que un usuario ya posee. Si el número de carpetas iFolder que posee un usuario ya excede el límite que se ha establecido, podrá seguir siendo el propietario de dichas carpetas iFolder.

Cuota de disco

Especifica el espacio máximo que puede utilizar un sistema de iFolder. Los valores de configuración válidos para todo el sistema sustituirán a las directivas de usuario.

Deseleccione la casilla de verificación para inhabilitar la cuota válida para todo el sistema.

Seleccione la casilla de verificación para habilitar la cuota válida para todo el sistema y, a continuación, especifique la cuota de espacio total (en MB) para la cuenta de un usuario.

Si habilita una cuota para todo el sistema que es inferior al espacio total actual del usuario para los datos de iFolder, los datos de las cuotas de los usuarios dejan de sincronizarse hasta que se reducen por debajo del límite o hasta que la cuota se aumenta a un valor mayor que el espacio total consumido por el usuario.

La acción de habilitar o modificar una cuota para todo el sistema no afecta a las cuotas de usuarios individuales existentes. Toda cuota de usuario existente sustituirá a la cuota válida para todo el sistema, independientemente de si la cuota de usuario es inferior o superior a la de todo el sistema.

Valor por defecto: Inhabilitado, Sin límite

Tamaño de archivo

Especifica el tamaño máximo de archivo que puede sincronizarse.

Si se ha especificado una cuota, el límite de tamaño máximo de archivo vigente coincidirá con el de la cuota.

Deseleccione la casilla de verificación para inhabilitar la directiva de límite de tamaño máximo de archivo. Si se inhabilita la directiva, el valor que se aplicará será Sin límite.

Seleccione la casilla de verificación para habilitar la directiva de límite de tamaño máximo de archivo y, a continuación, especifique el tamaño de archivo máximo permitido en MB.

Si se ha especificado una cuota, el límite de tamaño máximo de archivo por defecto coincidirá con el de la cuota.

Para determinar el límite de tamaño de archivo adecuado para las carpetas iFolder de su entorno, tenga en cuenta los siguientes datos:

  • Uso previsto

  • Frecuencia de modificación de los archivos de mayor tamaño

  • El modo en que las aplicaciones que hacen uso de los archivos de mayor tamaño guardan los cambios realizados en los archivos (archivo completo o delta)

  • Frecuencia con la que se sincronizan los archivos de cada miembro

  • Número de usuarios que comparten una carpeta iFolder

  • Si los usuarios acceden a iFolder por la red local o a través de una conexión WAN o de Internet

  • Ancho de banda disponible medio y máximo

Aunque defina un valor muy elevado como límite de tamaño de archivo y no existan cuotas para limitar los tamaños de archivo, el límite práctico vendrá controlado por el sistema de archivos del equipo del usuario. Por ejemplo, los volúmenes FAT32 tienen un tamaño máximo de archivo de 4 GB menos 1 byte.

Valor por defecto: Inhabilitado, Sin límite

Archivos excluidos

Especifique si desea restringir los tipos de archivo que se sincronizarán por medio de filtros de exclusión.

Escriba una extensión de archivo y, a continuación, haga clic en Añadir para añadirlo a la lista.

Para excluir un tipo de archivo de la sincronización, seleccione la casilla de verificación junto al tipo de archivo en la lista de tipos de archivo restringidos y luego, haga clic en Denegar.

Para permitir la sincronización de un tipo de archivo, seleccione la casilla de verificación junto al tipo de archivo en la lista de tipos de archivo restringidos y luego, haga clic en Permitir.

Para suprimir un tipo de archivo de la lista de tipos de archivo restringidos, seleccione la casilla de verificación junto al tipo de archivo y luego, haga clic en Suprimir.

Sincronización

Especifica el intervalo mínimo de sincronización en minutos.

Si se habilita esta opción, especificará el intervalo mínimo para sincronizar los datos de iFolder de cada cuenta de usuario. Cuanto mayor sea el valor, más restrictivo será.

Si esta opción está inhabilitada, el valor por defecto es de 5 minutos.

Cifrado

Especifica la directiva de cifrado para el sistema de iFolder. Los valores de configuración válidos para todo el sistema sustituirán a las directivas de usuario.

Activo
Seleccione Activo para habilitar la función de cifrado para el sistema iFolder. Al hacerlo, los usuarios podrán definir una directiva de cifrado para sus carpetas iFolder.

Deseleccione Activo para inhabilitar la función de cifrado para todos los usuarios.

Enforced (Aplicada)
Seleccione Aplicado para habilitar la función de cifrado para todos los usuarios. Cuando la directiva de cifrado se establece como Aplicada, el usuario no puede modificar los valores de cifrado de sus carpetas iFolder.

Compartimiento

 Activo
Por defecto, la función de compartición de iFolder está habilitada. Deseleccione Activo para inhabilitar la función de compartición del sistema iFolder. Una vez aplicada esta directiva, los usuarios de este sistema iFolder no pueden compartir las carpetas iFolder con otros usuarios. Sin embargo, es posible modificar los valores de directiva en el nivel de usuario o de grupo LDAP.

Aplicar
Seleccione Aplicar para aplicar la directiva de compartición definida para todo el sistema. Es posible aplicar tanto la habilitación como la inhabilitación de la función de compartición. Cuando se aplica la inhabilitación de la función de compartición, los valores de directiva para compartir en el nivel de usuario e iFolder se inhabilitan de forma automática y no es posible modificar los valores. No obstante, no está permitido definir la directiva para la opción Revocar.

    IMPORTANTE: no es posible habilitar esta opción a menos que se establezca la opción Compartimiento como Activo.


Revocar
Seleccione Revocar para eliminar los miembros compartidos de todas las carpetas iFolder del sistema iFolder.

Administrador secundario

La pestaña Administrador secundario permite añadir, editar o suprimir la información del administrador secundario.

Para añadir un administrador secundario:

  1. Haga clic en la pestaña Administrador secundario y, a continuación, haga clic en Añadir para ver la lista de usuarios iFolder.
  2. Seleccione el usuario que desea designar como administrador secundario y haga clic en Siguiente.
  3. Para asignar un grupo al administrador secundario, seleccione una opción de la lista Seleccionar grupo. Si el grupo seleccionado tiene establecido el límite de cuota de disco en conjunto, se rellenará el campo Fijar el límite de cuota de disco en conjunto para todo el grupo con ese valor. De lo contrario, el campo quedará vacío.
  4. Fijar los derechos de directiva para el administrador secundario.

Este recuadro de verificación está seleccionado por defecto. Para denegar este derecho al administrador secundario, deseleccione la casilla de verificación Dar permiso para suprimir iFolders.

Para editar la información del administrador secundario:

  1. Haga clic en la pestañaSecundario para mostrar la información del administrador secundario.

  2. Seleccione un administrador secundario y haga clic en Editar para mostrar la lista de grupos supervisados por el administrador secundario.

  3. Seleccione un grupo y haga clic en Editar para mostrar la lista de derechos del administrador secundario en el grupo. Edite los derechos del grupo y haga clic en Guardar para almacenar los cambios.
     

Para eliminar un administrador secundario:

  1. Haga clic en la pestañaSecundario para mostrar la información del administrador secundario.
  2. Seleccione un administrador secundario y haga clic en Eliminar para mostrar la lista de grupos supervisados por el administrador secundario.
  3. Seleccione todos los grupos y haga clic en Suprimir. Al suprimir todos los grupos propiedad del administrador secundario también se elimina el administrador secundario.
