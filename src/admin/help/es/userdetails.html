Información de usuario

Esta interfaz le permite consultar la información de usuario y gestionar las directivas de usuario.

NOTA: el término usuarios de iFolder hace referencia tanto a usuarios individuales como a grupos LDAP.

Información de usuario

La página Información de usuario muestra la siguiente información de usuario para la cuenta de iFolder del usuario seleccionado.

Parámetro

Descripción

Usuario

Nombre de usuario asignado a la cuenta de usuario como, por ejemplo, jsmith o john.smith@ejemplo.com.

Nombre completo

Nombre y apellido de la cuenta de usuario.

Tipo de miembro

Muestra el tipo de miembro del usuario que está conectado. Si se trata de un usuario individual, la interfaz también muestra una opción para grupos de usuarios. Si el usuario es miembro de un grupo LDAP, la interfaz muestra todos los miembros del grupo LDAP en la opción Miembros del grupo.

Contextos LDAP

El contexto del árbol LDAP permite asignar usuarios a iFolder.

Última entrada

Hora en la que el usuario entró por última vez en el sistema de iFolder.

Cuota de disco de grupo Representa la cuota de disco en conjunto de un grupo.

NOTA: el administrador secundario no puede modificar la cuota de disco de grupo.

Espacio de disco utilizado Indica el espacio de disco en conjunto utilizado por un grupo.

Grupos de usuarios (aplicable sólo a usuarios individuales)

Muestra una lista de todos los grupos a los que pertenece el usuario seleccionado.

Miembros del grupo (aplicable sólo a grupos LDAP)

Muestra una lista de todos los miembros que pertenecen al grupo LDAP seleccionado.

Carpetas iFolder

La página Información de usuario muestra las carpetas iFolder en propiedad o compartidas por el usuario. Haga clic en la pestaña Todos para obtener una lista de todas las carpetas iFolder, tanto compartidas como en propiedad. Para ver la carpeta iFolder en propiedad del usuario, haga clic en la pestaña En propiedad. La pestaña Compartido enumera las carpetas iFolder compartidas correspondientes a la cuenta de usuario seleccionada.

Habilitación o inhabilitación de una carpeta iFolder

  1. Localice la carpeta iFolder que desea gestionar y, a continuación, seleccione la casilla de verificación situada junto a la misma.

  2. Haga clic en Habilitar para habilitar la carpeta iFolder.

    Esta acción permitirá al usuario entrar en las carpetas iFolder y sincronizarlas.

  3. Haga clic en Inhabilitar para inhabilitar la carpeta iFolder.

    Si el usuario está conectado cuando se efectúa este cambio, la sesión del usuario continuará hasta que éste salga. La directiva surtirá efecto la próxima vez que el usuario intente entrar a la cuenta. Para que el bloqueo se aplique de forma inmediata, deberá reiniciar los servicios de Apache correspondientes al servidor iFolder, el cual desconectará todas las sesiones activas, incluida la del usuario.

Supresión de una carpeta iFolder

  1. Localice la carpeta iFolder que desea suprimir y, a continuación, seleccione la casilla de verificación situada junto a la misma.

  2. Haga clic en Suprimir.

Creación de una carpeta iFolder

  1. En la página Información de usuario, haga clic en Nuevo.

  2. En la página nueva que aparece, siga las instrucciones que se indican para crear una carpeta iFolder nueva.

Directivas

Esta interfaz le permite gestionar las directivas de usuario de una cuenta de usuario determinada.

NOTA: las directivas definidas para un grupo LDAP son aplicables a los usuarios que pertenecen al mismo.

Modificación de directivas de usuario

En la tabla siguiente se enumeran las directivas que pueden gestionarse para una cuenta de usuario de iFolder determinada. Modifique uno o varios de los valores siguientes y, a continuación, haga clic en Guardar para aplicar la nueva configuración:

Parámetro

Descripción

Cuenta

Especifica si el usuario tiene permiso actualmente para entrar en carpetas iFolder y sincronizarlas.

Seleccione la casilla de verificación para inhabilitar la entrada a la cuenta.

Deseleccione la casilla de verificación para habilitar la entrada a la cuenta.

Si el usuario está conectado cuando se efectúa este cambio, la sesión del usuario continuará hasta que éste salga. La directiva surtirá efecto la próxima vez que el usuario intente entrar en la cuenta. Para que el bloqueo se aplique de forma inmediata, deberá reiniciar los servicios de Apache correspondientes al servidor iFolder, el cual desconectará todas las sesiones activas, incluida la del usuario.

Valor por defecto: Habilitado

Número de carpetas iFolder por usuario

Especifica el número máximo de carpetas iFolder que puede poseer un usuario. Tras aplicar esta directiva, el usuario está limitado a poseer un determinado número de carpetas iFolder. El usuario que supere este límite recibirá un mensaje de error sobre la infracción de la directiva. Si el límite es cero, el usuario no puede crear ninguna carpeta iFolder.

Seleccione la casilla de verificación situada junto a Límite para habilitar el límite carpetas iFolder por usuario, y especifique el número en el campo.

El valor de configuración de la directiva no afecta al número de carpetas iFolder que el usuario ya posee. Si el número de carpetas iFolder que posee el usuario ya excede el límite que se ha establecido, podrá seguir siendo el propietario de dichas carpetas iFolder.

La directiva en el nivel de usuario anula la directiva en el nivel de sistema y de grupo LDAP.

Valor por defecto: Inhabilitado, sin valor establecido

Cuota de disco

Especifica el espacio total máximo que tienen permitido utilizar los datos de iFolder de un usuario para todas las carpetas iFolder que posea el usuario. La cuota puede venir impuesta por una directiva válida para todo el sistema. Las directivas de usuario sustituyen a las directivas válidas para todo el sistema.

Límite
Especifica el espacio total máximo (en MB) que tienen permitido utilizar los datos de iFolder de un usuario para todas las carpetas iFolder que posea el usuario y que correspondan a la cuenta del usuario seleccionado.

Deseleccione esta opción si no existe una cuota de usuario individual, o bien para aceptar la cuota del nivel de grupo LDAP o la cuota válida para todo el sistema para la cuenta de usuario seleccionada.
Seleccione esta opción para aplicar una cuota de usuario y, a continuación, especifique la cuota de espacio total (en MB) correspondiente a la cuenta de usuario seleccionada.

Usado
Especifica el espacio total en uso del servidor para todas las carpetas iFolder que el usuario seleccionado tiene en propiedad.

Disponibles
Espacio libre disponible del servidor.

Efectivo
Espacio real asignado en el servidor.

Si el espacio total consumido por los datos de iFolder se acerca a la cuota efectiva (sistema, grupo LDAP, usuario o iFolder), el usuario deberá detener la sincronización de los archivos hasta que una o más de las siguientes tareas permita obtener suficiente espacio para sincronizar, de forma segura, los archivos del usuario de la carpeta iFolder en la que reside el archivo:

  • La cuota válida para todo el sistema, la cuota del grupo LDAP, la cuota de usuario del propietario de la carpeta iFolder y la cuota de iFolder pueden modificarse según sea necesario.

  • Los archivos se mueven de cualquiera de las carpetas iFolder propiedad del usuario a otra ubicación en la que ya no afecten a la cuota efectiva, o bien se suprimen archivos para crear espacio.

IMPORTANTE: si habilita un límite de espacio de usuario inferior al espacio total actual de un usuario para los datos de iFolder, los datos del usuario dejarán de sincronizarse hasta que el volumen de datos se sitúe por debajo del límite o hasta que se aumente la cuota hasta un valor superior al espacio total consumido por el usuario.

Valor por defecto: Inhabilitado sin cuotas definidas

Tamaño del archivo

 Límite
Especifica el tamaño total máximo de archivo (en MB) que tiene permitido utilizar un usuario de iFolder para todas las carpetas iFolder que posea y que correspondan a la cuenta del usuario seleccionado.

Efectivo
Tamaño de archivo efectivo que tiene asignado el usuario.

    IMPORTANTE: los usuarios no podrán sincronizar correctamente archivos cuyo tamaño provoque un exceso de la cuota. Si lo intentan, sólo se sincronizará parte del archivo, por lo que los datos quedarán dañados.

Valor por defecto: Inhabilitado y sin límite

Archivos excluidos

Especifica una lista de los tipos de archivo que deben incluirse o excluirse de la sincronización para la cuenta de usuario seleccionada.

Los archivos del gestor de archivos thumbs.db y .DS_Store nunca se sincronizan. No es necesario conservar estos archivos, además, su sincronización provoca errores de conflicto de archivos repetidos. Si no ha definido ninguna restricción concreta para este usuario, el campo incluirá thumbs.db y .DS_Store como parte de las restricciones de tipo de archivo válidas para todo el sistema. Si, por el contrario, ha definido restricciones concretas de tipo de archivo para el usuario, se visualizarán en su lugar los valores de configuración del usuario. Aunque no se muestren las restricciones de thumbs.db y .DS_Store, éstas siempre se aplicarán, por lo que no es posible anularlas.

Para añadir una extensión de archivo a un filtro inclusión o exclusión, escriba la extensión (por ejemplo, *.mpg) y, a continuación, haga clic en Añadir para aplicar el filtro.

Para excluir un tipo de archivo de la sincronización, seleccione la casilla de verificación junto al tipo de archivo en la lista de tipos de archivo restringidos y luego, haga clic en Denegar.

Para permitir la sincronización de un tipo de archivo, seleccione la casilla de verificación junto al tipo de archivo en la lista de tipos de archivo restringidos y luego, haga clic en Permitir.

Para suprimir un tipo de archivo de la lista de tipos de archivo restringidos, seleccione la casilla de verificación junto al tipo de archivo y luego, haga clic en Suprimir.

Valor por defecto: .DS_Store y Thumbs.db

Sincronización

Especifica el intervalo mínimo de sincronización en segundos. Es posible definir un intervalo de sincronización en un valor situado entre 5 y 999, ambos incluidos. Si no se define ningún valor, se aplicará el valor 5.

Deseleccione la casilla de verificación si no desea definir ningún intervalo de sincronización o para aceptar la configuración válida para todo el sistema de la cuenta de usuario. Si no se define ningún valor para las directivas de usuario y válida para todo el sistema, se aplicará el valor Sin límite.

Intervalo
Seleccione la casilla de verificación para habilitar un intervalo de sincronización mínimo y, a continuación, especifique el intervalo mínimo (en minutos). Por ejemplo, un valor práctico puede ser de 10 minutos.

Efectivo
Especifica el intervalo de sincronización actual. Por ejemplo, si el usuario establece un intervalo de sincronización inferior (más frecuente) al mínimo del sistema, se aplicará el valor del sistema.

El intervalo de sincronización mínimo efectivo siempre corresponde al valor máximo de entre los siguientes:

  • La directiva del sistema (el valor por defecto es cero), a menos que se haya definido una directiva de usuario o de grupo LDAP. Si se ha definido una directiva de usuario, ésta anulará la directiva de sistema y de grupo LDAP, independientemente de si el valor del intervalo del usuario es superior o inferior.

  • La directiva del equipo local o el ajuste del equipo cliente que se sincroniza con el servidor.

  • La directiva de iFolder (recopilación).

Valor por defecto: Inhabilitado o el de la directiva válida para todo el sistema

Cifrado

Especifica la directiva de cifrado correspondiente a la cuenta de usuario seleccionada. La cuota puede venir impuesta por la directiva del sistema (válida para todo el sistema). La directiva válida para todo el sistema sustituye a las directivas de usuario.

Activo
Seleccione Activo para habilitar la función de cifrado para el usuario seleccionado. Al hacerlo, el usuario podrá definir una directiva de cifrado para sus carpetas iFolder.

Deseleccione Activo para inhabilitar la función de cifrado para el usuario seleccionado.

Enforced (Aplicada)
Seleccione Aplicado para habilitar la función de cifrado para el usuario seleccionado. Cuando se establece la opción Aplicado, el usuario no puede cambiar la configuración de cifrado para sus carpetas iFolder.

IMPORTANTE: esta opción sólo se habilitará si se ha activado la directiva de cifrado para el sistema.

Compartimiento

 Activo
Por defecto, la función de compartición de iFolder está habilitada. Deseleccione Activo y haga clic en Guardar para inhabilitar la función de compartición para el usuario seleccionado. Tras aplicar esta directiva, el usuario seleccionado no puede compartir las carpetas iFolder con otros usuarios. No obstante, aún es posible modificar la directiva en el nivel de iFolder.

Aplicar
Seleccione Aplicar para aplicar la directiva definida para el usuario seleccionado. Tras aplicar esta directiva, el usuario no puede compartir las carpetas iFolder.

Revocar
Seleccione Revocar para eliminar los miembros compartidos de todas las carpetas iFolder que pertenecen al usuario seleccionado.
