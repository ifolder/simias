Visualización de la información del servidor

Se puede ver y editar la información siguiente:

Parámetro

Descripción

Nombre (sólo visualización)

Nombre asignado al servidor.

Escriba

Tipo del servidor (Principal o Esclavo). Haga clic en Definir como principal para designar este servidor como servidor principal.

Nombre DNS

Nombre DNS del servidor empresarial de iFolder.

Por ejemplo, 192.168.1.1 o svr1.dominio.com.

URL pública

Dirección IP pública correspondiente al servidor iFolder. Para cambiar la dirección IP, edítela y haga clic en Guardar para guardar los cambios que haya realizado.

URL privada

Dirección IP privada correspondiente al servidor iFolder. Permite la comunicación entre los servidores del dominio iFolder. La URL privada puede coincidir con la URL pública. Para cambiar la dirección IP, escriba la nueva dirección y haga clic en Guardar para guardar los cambios que haya realizado.

URL principal (sólo se muestra para los servidores esclavos)

La dirección IP que corresponde al servidor iFolder. Utilizando esta dirección, el servidor esclavo se comunica con el servidor principal en el dominio de iFolder.

Para cambiar la dirección IP, edítela y haga clic en Guardar para guardar los cambios que haya realizado.

Definición de los niveles de registro

Es posible definir la siguiente información sobre el nivel del registro:

Parámetro

Descripción

Todos

Muestra todas las actividades del servidor que permiten al servicio de asistencia técnica de Novell resolver posibles problemas.

Depurar

Muestra las actividades del servidor que permiten al servicio de asistencia técnica de Novell depurar posibles problemas.

Información

Muestra las actividades básicas del servidor que permiten al servicio de asistencia técnica de Novell resolver posibles problemas. Esta opción está seleccionada por defecto.

Advertir

Muestra todos los posibles errores del sistema.

Error

Muestra todos los errores del sistema que impiden el funcionamiento correcto del mismo.

Fatal

Muestra los errores graves del sistema.

Inactivo

Se desactiva el registro.

Sincronización de identidad

Actualiza a los usuarios de iFolder del dominio iFolder seleccionado a partir de la información de LDAP correspondiente al intervalo seleccionado.

Especifique el intervalo en minutos en el campo Sincronización de identidad y haga clic en Sincronizar ahora para iniciar la sincronización inmediata de los usuarios de iFolder con los usuarios de LDAP.

Suprimir intervalo de gracia del miembro

Especifica el intervalo tras el cual iFolder suprimirá la información del usuario del servidor una vez que dicho usuario se haya suprimido de LDAP. Esta función permite recuperar los datos en caso de que el usuario se haya suprimido accidentalmente.


Es posible editar la siguiente información relacionada con LDAP. Haga clic en Editar para abrir una nueva página en la que se pueden modificar las entradas.

Parámetro

Descripción

Servidor LDAP

Muestra la dirección del servidor LDAP.

LDAP SSL

Indica si SSL LDAP está habilitado o no. También permite habilitar o inhabilitar la conexión SSL LDAP.

Usuario apoderado

Muestra el nombre completo (DN) del usuario apoderado. Por ejemplo, cn=SimiasProxy,o=novell

El usuario apoderado de iFolder es la identidad que se utiliza para acceder al servidor LDAP con el fin de recuperar una lista de los usuarios de los contenedores y grupos especificados, o bien de los usuarios definidos en los valores de configuración LDAP de iFolder. El usuario apoderado de iFolder se crea durante la instalación de iFolder. Esta identidad dene dispone del derecho de Lectura sobre el directorio LDAP.

Contraseña del apoderado

 

La contraseña se utiliza para autenticar al usuario apoderado de iFolder en el servidor LDAP cuando iFolder lleva a cabo la sincronización de los usuarios con el servidor LDAP.

Contexto LDAP

Este campo enumera todos los contextos LDAP. iFolder sólo busca los usuarios de la lista de contextos LDAP.


Almacén de datos

El almacén de datos representa el almacenamiento de iFolder capaz de extenderse por diferentes volúmenes (puntos de montaje) de un servidor determinado. Cada servidor iFolder dispone de un almacén de datos por defecto que no puede inhabilitarse. Mediante la interfaz Web, podrá añadir y configurar varios almacenes de datos en los que equilibrar la carga de los datos de iFolder. Cuando el usuario carga una carpeta iFolder, se comprueba el almacén de datos que dispone de mayor espacio libre y se almacenan los datos de la carpeta iFolder en dicho almacén, equilibrando de este modo la carga. Es posible añadir tantos almacenes de datos como se desee. La existencia de varios almacenes de datos permite escalar la capacidad de almacenamiento de datos en una distribución de gran tamaño para satisfacer los requisitos de la empresa.

Es posible consultar la siguiente información de almacenes de datos:

Parámetro

Descripción

Nombre

Muestra el nombre exclusivo especificado para el almacén de datos.

Ruta completa

Muestra la vía del almacén de datos en el que está montado el volumen. Se trata de la vía de datos que especificó al añadir el almacén de datos mediante la interfaz Web.

Espacio libre

Muestra el espacio disponible en el volumen.

Habilitada

Muestra si el almacén de datos en cuestión está habilitado o no. El almacén de datos por defecto no puede inhabilitarse.

Habilitación o inhabilitación de un almacén de datos

Seleccione el almacén de datos que desea habilitar o inhabilitar y haga clic en Habilitar o Inhabilitar, respectivamente. Cuando el usuario carga una carpeta iFolder, los almacenes de datos inhabilitados siempre se omiten durante la comprobación del almacén con la máxima cantidad de espacio libre para almacenar los datos de la carpeta iFolder.
