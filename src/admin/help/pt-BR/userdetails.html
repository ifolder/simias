Detalhes do usuário

Use esta interface para ver os detalhes do usuário e gerenciar as políticas deo usuário.

NOTA: O termo usuários do iFolder refere-se tanto aos usuários individuais quanto aos grupos LDAP.

Detalhes do usuário

A página Detalhes do Usuário exibirá os seguintes detalhes do usuário para a conta do iFolder do usuário selecionado.

Parâmetro

Descrição

Nome de usuário

O nome de usuário designado à conta do usuário, como jsilva ou jorge.silva@exemplo.com.

Nome completo

O nome e o sobrenome da conta do usuário.

Tipo de Membro

Mostra que tipo de membro é o usuário que efetuou login. Se o usuário for um usuário individual, a interface também exibirá uma opção para Grupos de Usuários. Se o usuário for membro de um grupo LDAP, a interface listará todos os membros do grupo LDAP sob a opção para Membros do Grupo.

Contextos LDAP

O contexto de árvore do LDAP é usado para os usuários de aprovisionamento no iFolder.

Último login

A hora em que o usuário efetuou o último login no sistema do iFolder.

Quota de discos do grupo Representa a quota de discos agregados de um grupo.

NOTA: O administrador secundário não pode modificar a quota de discos do grupo.

Espaço em Disco Utilizado Indica o espaço em disco agregado utilizado por um grupo.

Grupos de Usuários (aplica-se somente a usuários individuais)

Lista todos os grupos aos quais o usuário selecionado pertence.

Membros do Grupo (aplica-se apenas a grupos LDAP)

Lista todos os membros que pertencem ao grupo LDAP selecionado.

Pastas do iFolder

A página Detalhes do Usuário exibe os iFolders pertencentes ou compartilhados pelo usuário. Clique na guia Todos para listar todos os iFolders pertencentes e compartilhados. Para ver o iFolder pertencente ao usuário, clique na guia Pertencente. A guia Compartilhado lista todos os iFolders compartilhados para a conta do usuário atual.

Habilitando ou desabilitando um iFolder

  1. Localize o iFolder que você deseja gerenciar e marque a caixa de seleção ao lado do iFolder.

  2. Clique em Habilitar para habilitar o iFolder.

    Isso permitirá que o usuário faça login e sincronize iFolders.

  3. Clique em Desabilitar para desabilitar o iFolder.

    Se o usuário estiver conectado quando você fizer a mudança, a sessão do usuário continuará até o usuário se desconectar. A política fará efeito na próxima vez em que o usuário tentar efetuar login na conta. Para que o bloqueio faça efeito imediatamente, é preciso reiniciar os serviços Apache do servidor iFolder, que desconecta todas as sessões ativas, inclusive a sessão do usuário.

Apagando um iFolder

  1. Localize o iFolder que você deseja apagar e marque a caixa de seleção ao lado do iFolder.

  2. Clique em Apagar.

Criando uma pasta iFolder

  1. Na página Detalhes do Usuário, clique em Novo.

  2. Na nova página aberta, siga as instruções para criar um novo iFolder.

Políticas

Use esta interface para gerenciar as políticas de usuário de uma conta de usuário específica.

NOTA: As políticas definidas a um Grupo LDAP aplicam-se aos usuários membros.

Modificando políticas de usuário

A tabela a seguir lista as políticas que você pode gerenciar para qualquer conta de usuário do iFolder fornecida. Modifique um ou mais dos seguintes valores e clique em Gravar para aplicar as novas configurações.

Parâmetro

Descrição

Conta

Especifica se o usuário tem permissão atualmente para efetuar login em iFolders sincronizados.

Selecione a caixa de seleção para desabilitar a conta para login.

Desmarque a caixa de seleção para habilitar a conta para login.

Se o usuário estiver conectado quando você fizer essa mudança, a sessão do usuário continuará até ele se desconectar. A política fará efeito na próxima vez em que o usuário tentar efetuar login na conta. Para que o bloqueio faça efeito imediatamente, é preciso reiniciar os serviços Apache do servidor iFolder, que desconecta todas as sessões ativas, inclusive a sessão do usuário.

Valor Padrão: Habilitado

Número de pastas do iFolder por usuário

Especifica o número máximo de pastas do iFolder que um usuário pode possuir. Após a aplicação dessa política, o usuário fica limitado a um determinado número de pastas do iFolder. O usuário que exceder o limite receberá uma mensagem de erro sobre a violação da política. Se o limite for zero, o usuário não poderá criar nenhuma pasta do iFolder.

Marque a caixa de seleção ao lado de Limitar para habilitar o limite por usuário do iFolder e especifique o número no campo.

A configuração da política não afeta o número de pastas do iFolder que o usuário já possui. Se o número de pastas do iFolder do usuário já excede o limite definido, ele mesmo assim ainda pode mantê-las.

A política no nível do usuário substitui a no nível do grupo LDAP e do sistema.

Valor Padrão: Desabilitado, sem valor definido

Quota de discos

Especifica o espaço máximo total que os dados do iFolder de um usuário podem usar em todos os iFolders pertencentes ao usuário. A quota pode ser imposta pela política no âmbito do sistema. As políticas de usuário substituem as políticas no âmbito do sistema.

Limite
Especifica o espaço máximo total (em MB) que os dados do iFolder de um usuário podem usar em todos os iFolders pertencentes ao usuário para a conta de usuário selecionada.

Desmarque essa opção se não houver quota individual de usuário ou para aceitar a quota no nível do grupo LDAP ou no âmbito do sistema para a conta de usuário selecionada.
Selecione esta opção para forçar uma cota de usuário e especifique a cota de espaço total (em MB) para a conta de usuário selecionada.

Usado
Especifica o espaço total em uso atualmente no servidor para todos os iFolders pertencentes a esse usuário selecionado.

Disponível
Espaço livre disponível no servidor.

Efetivo
Espaço efetivo alocado no servidor

Se o espaço total consumido por dados do iFolder estiver próximo de uma quota efetiva (sistema, grupo LDAP, usuário ou iFolder), o usuário deverá interromper a sincronização dos arquivos até que uma ou mais das seguintes tarefas resultem em espaço suficiente para sincronizar, com segurança, os arquivos do usuário no iFolder em que o arquivo reside:

  • As quotas de todo o sistema, do grupo LDAP e do usuário para o proprietário do iFolder e a quota do iFolder são modificadas conforme necessário.

  • Os arquivos são movidos de uma das pastas do iFolder pertencentes ao usuário para outro local onde elas não afetem mais a quota efetiva, ou os arquivos são apagados para liberar espaço.

IMPORTANTE:Se você habilitar um limite de espaço de usuário que seja menor que o espaço total atual de um usuário para dados do iFolder, os dados do usuário não serão mais sincronizados até que a quantidade de dados seja diminuída para um valor abaixo do limite ou até que a cota seja aumentada para um valor maior que o espaço total consumido pelo usuário.

Valor padrão: Desabilitado e sem quota definida

Tamanho do arquivo

 Limite
Especifica o tamanho máximo total do arquivo (em MB) ao qual um usuário do iFolder tem permissão de uso, em todos os iFolders que o usuário possui, para a conta de usuário selecionada.

Efetivo
Tamanho efetivo do arquivo alocado para o usuário.

    IMPORTANTE:Os usuários não podem sincronizar arquivos de um tamanho que faça com que a quota seja excedida. Caso eles tentem fazer isso, apenas parte do arquivo será sincronizada, o que resultará na corrupção dos dados.

Valor padrão: Desabilitado e sem limite

Arquivos excluídos

Especifica uma lista de tipos de arquivos para incluir ou excluir da sincronização da conta do usuário selecionado.

Os arquivos do gerenciador de arquivos chamados thumbs.db e .DS_Store nunca são sincronizados. Você não precisa manter esses arquivos e a sincronização dos mesmos resulta em erros repetidos de conflito de arquivos. Se você não tiver definido restrições individuais para esse usuário, esse campo relatará thumbs.db e .DS_Store como parte das restrições de tipo de arquivo no âmbito do sistema. Depois de definir as restrições individuais de tipo de arquivo para o usuário, serão exibidas as configurações do usuário. Mesmo se as restrições thumbs.db e .DS_Store não forem exibidas, elas sempre se aplicarão; você não pode substituí-las.

Para adicionar uma extensão de arquivo a um filtro de inclusão ou exclusão, digite a extensão (como *.mpg) e clique em Adicionar para aplicar o filtro.

Para excluir um tipo de arquivo da sincronização, marque a caixa de seleção adjacente ao tipo de arquivo na lista de tipos de arquivos restritos, então clique em Negar.

Para permitir que um tipo de arquivo seja sincronizado, marque a caixa de seleção adjacente ao tipo de arquivo na lista de tipos de arquivos restritos, então clique em Permitir.

Para apagar um tipo de arquivo da lista de tipos de arquivos restritos, marque a caixa de seleção adjacente ao tipo de arquivo e clique em Apagar.

Valor padrão: .DS_Store e Thumbs.db

Sincronização

Especifique o intervalo mínimo de sincronização em segundos. Você pode definir o intervalo de sincronização como um valor entre 5 e 999 inclusive. Se nenhum valor for definido, o valor será 5.

Desmarque a caixa de seleção para não definir intervalo de sincronização ou para aceitar a configuração no âmbito do sistema da conta do usuário. Se não houver um valor definido para âmbito do sistema ou políticas do usuário, o valor relatado será Ilimitado.

Intervalo
Marque a caixa de seleção para habilitar um intervalo mínimo de sincronização e especifique o intervalo mínimo (em minutos). Por exemplo, um valor prático é 10 minutos.

Efetivo
Especifica o intervalo de sincronização atual. Por exemplo, se o usuário definir um intervalo de sincronização que seja menor (mais freqüente) que o intervalo mínimo do sistema, a configuração do sistema será aplicada.

O intervalo mínimo de sincronização efetivo é sempre o valor maior das seguintes configurações:

  • A política do sistema (padrão igual a zero), a não ser que exista uma política do grupo LDAP ou um conjunto de políticas do usuário definido. Se uma política de usuário for definida, ela substituirá a política do sistema e a do grupo LDAP, seja o intervalo definido pelo usuário mais curto ou mais longo.

  • A política da máquina local ou a configuração na máquina do cliente sincronizando com o servidor.

  • A política do iFolder (coleção).

Valor padrão: Desabilitado ou política no âmbito do sistema

Criptografia

Especifica a política de criptografia para a conta do usuário selecionado. A cota pode ser imposta pela política do sistema (âmbito do sistema). A política no âmbito do sistema substitui políticas de usuário.

Ativar
Selecione Ativar para habilitar o recurso de criptografia para o usuário selecionado. Isso permite que o usuário defina a política de criptografia para seus iFolders.

Desmarque Ativar para desabilitar o recurso de criptografia para o usuário selecionado.

Obrigatório
Selecione Obrigatório para habilitar o recurso de criptografia para o usuário selecionado. Quando estiver definido como Obrigatório, o usuário não poderá mudar as configurações de criptografia para iFolders.

IMPORTANTE: Essa opção está habilitada somente se a política de criptografia estiver definida como Ativar.

Compartilhar

 Ativar
Por padrão, o compartilhamento do iFolder fica habilitado. Desmarque a caixa de seleção Ativar e clique em Gravar para desabilitar o compartilhamento para o usuário selecionado. Após a aplicação dessa política, o usuário selecionado não poderá compartilhar suas pastas do iFolder com outras pessoas. Entretanto, ainda é possível mudar a política no nível do iFolder.

Obrigatório
Selecione Obrigatório para assegurar o uso obrigatório do conjunto de políticas para o usuário selecionado. Após a aplicação dessa política, o usuário não poderá compartilhar suas pastas do iFolder.

Revogar
Selecione Revogar para remover os membros compartilhados de todas as pastas do iFolder pertencentes ao usuário selecionado.
