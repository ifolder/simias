Gerenciando o sistema do iFolder

Use esta interface para gerenciar os serviços do Novell iFolder® 3.7 para o servidor selecionado.

Esta Ajuda mostra as seguintes informações:

Configurações do sistema

Configurações do sistema

Parâmetro

Descrição

O nome atribuído ao domínio do iFolder.

Para editar o nome de domínio do iFolder, digite o novo nome e clique em Gravar.

Para cancelar as mudanças feitas, clique em Cancelar.

Descrição

Uma breve descrição do domínio do iFolder..

Para editar a descrição do sistema, digite a nova descrição e clique em Gravar.

Para cancelar as mudanças feitas, clique em Cancelar.

Opção SSL

Total de iFolders

Ordem de Exibição do Nome Completo

Permite que você configure a ordem na qual o nome completo de um usuário será exibido. Selecione a opção (Nome, Sobrenome) para exibir o nome seguido pelo sobrenome. Para exibir o sobrenome seguido pelo nome, selecione a opção (Sobrenome, Nome).

Para que as mudanças sejam efetuadas, programe ou execute manualmente uma sincronização LDAP. Para fazer uma sincronização LDAP manual:

  1. No console do Web admin, clique na guia Servidores, selecione o servidor e vá para a página Serverdetails.

  2. Na seção Detalhes do LDAP, clique no botão Sincronizar Agora.

Gerenciar Quota de Grupo Usando

Permite que você defina como o conjunto de quota de disco agregado será gerenciado.

Selecione a opção Console do Administrador para permitir que os administradores gerenciem a quota de discos de grupos ou membros de um grupo. Quando você selecionar essa opção, a quota de discos atribuída aos usuários será restringida para que não exceda o uso do disco agregado do grupo. A opção Console do Administrador é selecionada por padrão.

Selecione a opção Mecanismo de Sincronização para habilitar o mecanismo de sincronização para gerenciar a quota de discos agregados em grupos no back end. O mecanismo de sincronização garante que a quota de discos de todos os usuários em um grupo não exceda a quota de discos agregados do grupo.

Selecione a opção Ambos para usar o console do administrador e o mecanismo de sincronização para gerenciar o uso da quota de discos agregados dos grupos.

Grupos Segregados Permite segregar grupos em entidades independentes e garantir que os membros de um grupo não poderão ser acessados por membros de outro grupo para compartilhar iFolders. Marque a caixa de seleção Criar Grupos Segregados para permitir o compartilhamento de iFolders apenas entre membros do mesmo grupo.

Nota: Um Grupo LDAP não pode ser um Administrador Secundário ou Primário.

Status do Reprovisionamento

Você pode mover usuários entre servidores diferentes. Com esta interface, você pode ver o status do reprovisionamento. Clique em Status do Reprovisionamento para ver o status do reprovisionamento de cada usuário.

Administradores do iFolder

O usuário Admin do iFolder é o administrador principal do servidor empresarial iFolder. Sempre que os iFolders ficarem órfãos, a propriedade será transferida para o usuário Admin do iFolder para ser reatribuído a outro usuário ou ser apagado. Inicialmente, especifique o usuário Admin do iFolder durante a configuração do servidor empresarial iFolder no YaST.

O usuário Admin do iFolder deve ser provisionado para habilitar o Admin do iFolder a desempenhar tarefas de gerenciamento. O iFolder controla esse usuário através do GUID do objeto LDAP, permitindo assim que ele pertença a qualquer grupo ou container LDAP na árvore, mesmo aqueles que não são identificados como DNs de Pesquisa. O movimento do usuário pode ser controlado em qualquer lugar na árvore porque ele é conhecido pelo GUID e não pelo DN do usuário.

O Admin do iFolder pode ser designado a outros usuários de forma que eles também possam gerenciar serviços do iFolder para o servidor selecionado. Somente usuários que estejam em um dos DNs especificados no DN de Pesquisa LDAP são elegíveis para serem equivalentes ao usuário Admin do iFolder.

Se você designar o direito Admin do iFolder a outros usuários, esses usuários serão governados pelo relacionamento entre a pasta e o DN de Pesquisa. O usuário é removido da pasta e retirado do direito Admin do iFolder se você apagar o usuário, remova o DN do usuário da lista de DNs de Pesquisa ou mova o usuário para um DN que não esteja nos DNs de Pesquisa.

O usuário Admin do iFolder ou o administrador primário cria o administrador secundário e atribui grupos ao administrador secundário.

IMPORTANTE: Não é possível designar o direito do usuário Admin a um Grupo LDAP.

A página Sistema exibe as guias Administrador Primário e Secundário. Os parâmetros a seguir são comuns às duas guias:

Parâmetro

Tipo

Nome do Usuário

O nome do usuário designado à conta de usuário Admin, como jsilva ou jorge.silva@exemplo.com.

Nome completo

O nome e o sobrenome na conta do usuário Admin.

Administrador Primário

Para ver ou editar os detalhes do administrador primário, clique na guia Administrador Primário e no link Usuário Admin para abrir a página Detalhes do Usuário. A página Detalhes do Usuário exibe os iFolders que pertencem ou estão compartilhados pelo usuário Admin. Clique na guia Todos para listar todos os iFolders, pertencentes e compartilhados. Para ver o iFolder pertencente ao usuário, clique na guia Pertencente. A guia Compartilhado lista todos os iFolders compartilhados para essa conta de usuário específica. Você também pode usar a página Detalhes do Usuário para mudar as configurações de política do usuário Admin selecionado.

Adicionando e apagando o Admin do iFolder


  1. Localize o usuário Admin que você deseja gerenciar e marque a caixa de seleção ao lado dele.

    • Clique em Apagar para apagar o usuário Admin selecionado.

      Se você apagar o direito Admin do iFolder do nome de usuário usado ao efetuar login no servidor, você será imediatamente desconectado. Você deve efetuar login no servidor iFolder usando um nome de usuário diferente, com o direito Admin do iFolder, para continuar gerenciando o servidor.

      IMPORTANTE: Você não pode apagar o usuário Admin definido durante a configuração do servidor Simias.

    • Clique em Adicionar para abrir uma lista de usuários do iFolder.

      Localize o usuário que você deseja adicionar como Admin, marque a caixa de seleção ao lado dele e clique em Adicionar.

      Você pode designar o direito Admin do iFolder para vários usuários.

Políticas

A tabela a seguir lista políticas do sistema que você pode gerenciar para qualquer Sistema do iFolder. Clique em Gravar para aplicar as modificações:

Parâmetro

Descrição

iFolder por usuário

Especifica o número máximo de pastas do iFolder permitidas por usuário. Após a aplicação dessa política, cada usuário fica limitado a um determinado número de pastas do iFolder. Os usuários que excedem o limite recebem uma mensagem de erro sobre a violação da política. Se o limite for zero, os usuários não poderão criar nenhuma pasta do iFolder.

A configuração da política não afeta o número de pastas do iFolder que o usuário já possui. Se o número de pastas do iFolder de um usuário já excede o limite definido, ele mesmo assim ainda pode mantê-las.

Quota de disco

Especifica o espaço máximo que o sistema de um iFolder tem permissão para usar. As configurações no âmbito do sistema substituem as políticas do usuário.

Desmarque a caixa de seleção para desabilitar uma cota no âmbito do sistema.

Marque a caixa de seleção para habilitar uma cota no âmbito do sistema, especifique a cota total de espaço (em MB) para uma conta do usuário.

Se você habilitar uma quota no nível do sistema que seja menor do que o espaço total atual do usuário para os dados do iFolder, os dados das quotas dos usuários não serão mais sincronizados até que a quantidade de dados seja diminuída para um valor abaixo do limite ou até que a quota seja aumentada para um valor maior do que o espaço total consumido pelo usuário.

Habilitar ou modificar a cota no âmbito do sistema não afeta a existência das cotas de usuário individuais. Qualquer cota de usuário existente sempre substitui uma cota no âmbito do sistema, independentemente da cota de usuário ser menor ou maior que a cota no âmbito do sistema.

Valor padrão: Desabilitado, Sem limite

Tamanho do arquivo

Especifica o tamanho máximo de arquivo que pode ser sincronizado.

Se uma cota for especificada, o limite máximo de tamanho efetivo do arquivo será o mesmo que a cota.

Desmarque a caixa de seleção para desabilitar a política Limite Máximo de Tamanho do Arquivo. Se a política estiver desabilitada, o valor será relatado como Sem Limite.

Marque a caixa de seleção para habilitar a política Limite Máximo de Tamanho do Arquivo e especifique o tamanho máximo permitido do arquivo em MB.

Se uma cota for especificada, o limite máximo de tamanho do arquivo será o mesmo que a cota.

Considere as seguintes demandas do sistema para determinar um limite adequado de tamanho do arquivo para iFolders no seu ambiente:

  • Tipo de uso

  • Freqüência com que os arquivos maiores são modificados

  • Como os aplicativos que usam os arquivos maiores realmente gravam as mudanças no arquivo (o arquivo inteiro ou deltas)

  • Qual a freqüência com que os arquivos são sincronizados por cada membro

  • Quantos usuários compartilham um iFolder

  • Se os usuários acessam o iFolder na rede local ou através de WAN ou conexão com a Internet

  • A média e o pico da largura de banda disponível

Mesmo se você definir um valor muito alto como limite do tamanho do arquivo e se não houver cota para limitar os tamanhos dos arquivos, o limite prático será governado pelo sistema de arquivos no computador do usuário. Por exemplo, os volumes FAT32 têm o tamanho máximo de arquivo de 4 GB menos 1 byte.

Valor padrão: Desabilitado, Sem limite

Arquivos Excluídos

Especifique se é necessário restringir tipos de arquivos que são sincronizados por filtros de exclusão.

Digite a extensão de arquivo e clique em Adicionar para adicioná-lo à lista.

Sincronização

Especifica o intervalo mínimo de sincronização em minutos.

Se essa opção estiver habilitada, especifique o intervalo mínimo para a sincronização de dados do iFolder para cada conta do usuário. Valores maiores são mais restringíveis.

Se a opção estiver desabilitada, o valor padrão será 5 minutos.

Criptografia

Especifica a política de criptografia para o sistema do iFolder. As configurações no âmbito do sistema substituem as políticas do usuário.

Ativar
Selecione Ativar para habilitar o recurso de criptografia para o sistema iFolder. Isso permite que um usuário defina uma política de criptografia para suas pastas do iFolder.

Desmarque Ativado para desabilitar o recurso de criptografia de todos os usuários.

Obrigatório
Selecione Obrigatório para habilitar o recurso de criptografia de todos os usuários. Quando a política de criptografia estiver definida como Obrigatório, o usuário não poderá mudar as configurações de criptografia de suas pastas do iFolder.

Compartilhar

 Ativar
Por padrão, o compartilhamento do iFolder fica habilitado. Desmarque Ativar para desabilitar o compartilhamento do sistema do iFolder. Após aplicar essa política, os usuários desse sistema iFolder não poderão compartilhar suas pastas com outros usuários. Entretanto, você poderá mudar as configurações da política no nível do usuário ou do grupo LDAP.

Obrigatório
Selecione Obrigatóriopara assegurar o uso obrigatório da política de compartilhamento que você definiu para todo o sistema. Você pode assegurar o uso obrigatório da habilitação e da desabilitação do compartilhamento. Quando é assegurado o uso obrigatório da desabilitação do compartilhamento, as configurações da política de compartilhamento no nível do iFolder e do usuário são automaticamente desabilitadas, e não há permissão para mudar as configurações. Entretanto, você tem permissão para definir a política para a opção Revogar.

    IMPORTANTE: Você não poderá habilitar essa opção a menos que defina a opção Compartilhar como Ativar.


Revogar
Selecione Revogar para remover os membros compartilhados de todas as pastas sob o sistema iFolder.

Administrador Secundário

A guia Administrador Secundário permite adicionar, editar ou apagar os detalhes do administrador secundário.

Para adicionar um administrador secundário:

  1. Clique na guia Administrador Secundário e em Adicionar para exibir a lista de usuários do iFolder.
  2. Selecione o usuário que deseja designar como administrador secundário e clique em Avançar.
  3. Para atribuir um grupo ao administrador secundário, selecione uma opção na lista Selecionar Grupo. Se o grupo selecionado possuir um limite de quota de discos agregados, o campo Configurar o Limite de Quota de Discos Agregados de Todo o Grupo será preenchido com esse valor. Se não houver um limite, o campo permanecerá vazio.
  4. Configure os direitos de política do administrador secundário.

    A tabela a seguir lista os direitos de política que você pode configurar para o administrador secundário.

    Parâmetro

    Descrição

    Política por usuário do iFolder

     

    Especifica o número máximo de iFolders permitido por usuário. Após a aplicação dessa política, cada usuário fica limitado a um determinado número de pastas do iFolder. Os usuários que excedem o limite recebem uma mensagem de erro sobre a violação da política. Se o limite for zero, os usuários não poderão criar nenhuma pasta do iFolder.

    Essa configuração de política não afeta o número de iFolders que um usuário já possui. Se o número de iFolders de um usuário já exceder o limite que você configurou, o usuário ainda poderá ser proprietário desses iFolders.

    Por padrão, a caixa de seleção Permitir é selecionada para a política Por usuário do iFolder. Isso significa que o administrador secundário tem o direito de configurar a política por usuário do iFolder para os usuários do grupo designado. Para não conceder esse direito ao administrador secundário, desmarque a caixa de seleção Permitir.

    Política de quota em disco

    Especifica o espaço máximo que um usuário tem permissão para usar.

    Por padrão, a caixa de seleção Permitir é selecionada para a política de quota de discos. Isso significa que o administrador secundário tem o direito de configurar a política de quota de discos para os usuários do grupo designado. Para não conceder esse direito ao administrador secundário, desmarque a caixa de seleção Permitir.

    Política de Tamanho de Arquivo

    Especifica o tamanho máximo de arquivo que pode ser sincronizado.

    Por padrão, a caixa de seleção Permitir é selecionada para a política de tamanho de arquivo. Isso significa que o administrador secundário tem o direito de configurar a política de tamanho de arquivo para os usuários do grupo designado. Para não conceder esse direito ao administrador secundário, desmarque a caixa de seleção Permitir.

    Política de intervalo de sincronização

    Especifica o intervalo mínimo de sincronização em minutos.

    Por padrão, a caixa de seleção Permitir é selecionada para a política de intervalo de sincronização. Isso significa que o administrador secundário tem o direito de configurar a política de intervalo de sincronização para os usuários do grupo designado. Para não conceder esse direito ao administrador secundário, desmarque a caixa de seleção Permitir.

    Política da lista de arquivos excluídos

    Especifica os tipos de arquivos que não podem ser sincronizados.

    Por padrão, a caixa de seleção Permitir é selecionada para a política da lista de arquivos excluídos. Isso significa que o administrador secundário tem o direito de configurar a política da lista de arquivos excluídos para os usuários do grupo designado. Para não conceder esse direito ao administrador secundário, desmarque a caixa de seleção Permitir.

    Compartilhamento

    Especifica se iFolders podem ser compartilhados entre usuários.

    Por padrão, a caixa de seleção Permitir modificação da política de compartilhamento é selecionada para a política de compartilhamento. Isso significa que o administrador secundário tem o direito de modificar a política de compartilhamento dos usuários do grupo designado. Para não conceder esse direito ao administrador secundário, desmarque a caixa de seleção Permitir.

    Política de criptografia

    Especifica a política de criptografia para o sistema do iFolder.

    Por padrão, a caixa de seleção Permitir modificação da política de criptografia é selecionada para a política de criptografia. Isso significa que o administrador secundário tem o direito de modificar a política de criptografia dos usuários do grupo designado. Para não conceder esse direito ao administrador secundário, desmarque a caixa de seleção Permitir.

    Direitos de aprovisionamento

    Especifica os direitos de aprovisionamento disponíveis para um administrador secundário.

    A caixa de seleção Permitir aprovisionamento de usuário é selecionada por padrão. Isso significa que o administrador secundário pode aprovisionar os usuários do grupo designado para qualquer servidor presente na instalação com vários servidores do iFolder. Para não conceder esse direito ao administrador secundário, desmarque a caixa de seleção Permitir o aprovisionamento de usuários.

    A caixa de seleção Permitir habilitação/desabilitação de usuáriospermite que você atribua ao administrador secundário o direito de habilitar ou desabilitar usuários do grupo designado. Esta caixa de seleção é marcada por padrão. Para não conceder esse direito ao administrador secundário, desmarque a caixa de seleção Permitir habilitação/desabilitação de usuários.

    Direitos sobre iFolders

    Especifica direitos do administrador secundário sobre iFolders que pertencem a usuários do grupo designado.

    Para permitir que o administrador secundário seja o proprietário de iFolders órfãos, verifique se a caixa de seleção Permitir propriedade de iFolders órfãos está marcada. Esta caixa de seleção é marcada por padrão. Para não conceder esse direito ao administrador secundário, desmarque a caixa de seleção.

    A caixa de seleção Permitir Habilitação/Desabilitação de iFolders pode ser usada para atribuir a um administrador secundário o direito de habilitar ou desabilitar iFolders que pertencem a usuários do grupo designado. Esta caixa de seleção é marcada por padrão. Para não conceder esse direito ao administrador secundário, desmarque a caixa de seleção Permitir Habilitação/Desabilitação de iFolders.

    A caixa de seleção Permitir modificações nos direitos de membros do iFolder compartilhado pode ser usada para atribuir ao administrador secundário o direito de modificar os direitos dos membros do iFolder compartilhado. Esta caixa de seleção é marcada por padrão. Para não conceder esse direito ao administrador secundário, desmarque a caixa de seleção Permitir modificações nos direitos de membros do iFolder compartilhado .

 

  1. Clique no botão Gravar para gravar suas configurações.
  2. Após atribuir um grupo ao administrador secundário, clique em OK, para voltar à página Sistemas, ou em Repetir, para atribuir mais grupos ao administrador secundário. 

Para editar detalhes do administrador secundário:

  1. Clique na guia Secundário para exibir os detalhes do administrador secundário.

  2. Selecione um administrador secundário e clique em Editar para exibir a lista de grupos monitorados pelo administrador secundário.

  3. Selecione um grupo e clique em Editar para exibir a lista de direitos de administrador secundário do grupo. Edite os direitos do grupo e clique em Gravar para gravar as modificações.
     

Para apagar um administrador secundário:

  1. Clique na guia Secundário para exibir os detalhes do administrador secundário.
  2. Selecione um administrador secundário e clique em Apagar para exibir a lista dos grupos monitorados pelo administrador secundário.
  3. Selecione todos os grupos e clique em Apagar. Se todos os grupos de um administrador secundário forem apagados, o administrador secundário também será apagado.
