Управление подробной информацией о папке iFolder

Используйте этот интерфейс для управления владением, политиками и членством выбранной папки iFolder.

Примечание. Дополнительный администратор может управлять политиками выбранной папки iFolder на основании своих прав по отношению к этой папке.

Владелец

Имя пользователя владельца выбранной папки iFolder. Владельцем может быть индивидуальный пользователь или группа LDAPGroup. Если группа LDAPGroup является владельцем папки iFolder, у всех ее членов одинаковые права доступа. Владельцем папок iFolder с нарушенными ассоциациями назначается администратор iFolder до тех пор, пока они не будут переназначены или удалены.

Щелкните ссылку с именем пользователя владельца для просмотра подробной информации о его учетной записи.

Владелец папки iFolder обладает правом доступа "Полный контроль" для этой папки. Владелец управляет членством и правами доступа пользователей, а также может лишить любого из членов права "Полный контроль".

Путь

Действительное расположение папки iFolder и ее данных на сервере. Например:

/var/opt/novell/ifolder3/simias/SimiasFiles/e84fdc6e-3d51-49df-ae3f-8c9213c76994/<имя_iFolder>

В данном примере e84fdc6e-3d51-49df-ae3f-8c9213c76994 - это уникальный идентификатор совместного использования iFolder.

С нарушенными ассоциациями

Значение "Да" указывает на то, что для данной папки iFolder нарушены ассоциации.

Владельцем папок iFolder с нарушенными ассоциациями назначается администратор iFolder до тех пор, пока они не будут переназначены другому владельцу или удалены.

Члены

Вы можете просматривать имена членов, их тип и права доступа. Можно добавлять или удалять владельца, назначать владение и права доступа выбранному члену.

Добавление члена

  1. На странице "Информация iFolder" щелкните "Добавить".

  2. Найдите пользователя или группу LDAPGroup, которых необходимо сделать членами, установите флажок рядом с соответствующим именем, а затем щелкните "Добавить".

    По умолчанию пользователю или группе LDAPGroup предоставляется доступ "Только чтение" к папке iFolder.

  3. (Не обязательно) Установите флажок "Пользователь", выберите "Права" и установите для права "Доступ" значение "Администратор", "Чтение/запись" или "Только чтение", а затем щелкните "Установить".

    Дождитесь завершения обновления страницы. Значок пользователя должен отражать новые права доступа.

Общие сведения о правах доступа пользователей iFolder

Клиент iFolder поддерживает владельца и три уровня доступа для членов папки iFolder:

ПРИМЕЧАНИЕ. Члены группы LDAPGroup наследуют права доступа, установленные для данной группы LDAPGroup.

Владелец
Только один пользователь или одна группа LDAPGroup может быть владельцем папки iFolder. Обычно это пользователь или любой член группы LDAPGroup, создавший папку iFolder. Если владельцем папки iFolder является группа LDAPGroup, права на ее владение принадлежат всем членам этой группы LDAPGroup. Пользователь или группа LDAPGroup, владеющие папкой iFolder, могут передать права владения другому члену iFolder. Администратор iFolder может передавать право на владение любому пользователю или группе LDAPGroup на сервере iFolder.

Владелец папки iFolder имеет право на полный контроль. Этот пользователь или группа LDAPGroup имеют доступ для чтения/записи к папке iFolder, управляют членством и правами доступа пользователей и групп LDAPGroup, а также могут удалять право "Полный контроль" для любого члена. На корпоративном сервере дисковое пространство, занимаемое папками iFolder конкретного владельца, рассчитывается на основе его пользовательских дисковых квот на данном корпоративном сервере.

В случае удаления пользователя или группы LDAPGroup из системы iFolder папки iFolder, которыми владеет данный пользователь или группа LDAPGroup, становятся папками с нарушенными ассоциациями. Владельцем папок iFolder с нарушенными ассоциациями временно назначается администратор iFolder. Поддержка членства и синхронизация для других пользователей или членов группы LDAPGroup продолжаются, пока администратор iFolder не определит, нужно ли удалить папку iFolder с нарушенными ассоциациями или назначить ей другого владельца.

Полный контроль
Уровень доступа члена совместно используемой папки iFolder с правом на полный контроль. Пользователь или группа LDAPGroup с правами "Полный контроль" имеет доступ на чтение/запись к папке iFolder, управляет членством и правами доступа всех пользователей, за исключением владельца.

Для чтения/записи
Уровень доступа члена совместно используемой папки iFolder с правом на чтение/запись в каталогах и файлах папки iFolder.

Только чтение
Уровень доступа члена совместно используемой папки iFolder с правом только для чтения в каталогах и файлах папки iFolder. Этот член может скопировать файл из папки iFolder в другое место и изменить его вне папки iFolder.

Установка прав доступа iFolder для члена

  1. На странице "Информация iFolder" установите флажок рядом с членом, права доступа которого необходимо изменить.

  2. Выберите раскрывающееся меню "Права", затем выберите права "Администратор", "Чтение/запись" или "Только чтение".

    Дождитесь завершения обновления страницы. Значок пользователя должен отражать новые права доступа.

Удаление члена

  1. На странице "Информация iFolder" установите флажок рядом с именем пользователя члена.
  2. Щелкните "Удалить".

Локальная копия данных пользователя сохраняется на его компьютере, однако пользователь не может получить доступ к копии данных iFolder, хранящейся на сервере.

Изменение владельца

При изменении владельца папки iFolder предыдущий владелец становится членом этой папки iFolder с правами "Чтение/запись". Владельцем папок iFolder с нарушенными ассоциациями назначается администратор iFolder до тех пор, пока они не будут переназначены другому владельцу или удалены. Если новый владелец не является членом данной папки iFolder, в его учетную запись iFolder отправляется уведомление о подписке.

  1. На странице "Информация iFolder" выполните поиск пользователя или группы LDAPGroup, которых необходимо назначить новыми владельцами папки iFolder.

  2. Установите флажок рядом с его именем пользователя, а затем щелкните Владелец.

Размер файла

 Лимит
Максимальный общий размера файла (в Мб) для всех папок iFolder, которыми пользователь владеет в рамках данной учетной записи пользователя.

Действующий
Действительный размер файла, выделенный для пользователя.

    ВАЖНО. Пользователи не могут выполнить синхронизацию файлов, размер которых нарушает квоту. Если они попытаются сделать это, файл синхронизируется лишь частично, что вызывает повреждение данных.

Исключенные файлы

Список типов файлов, разрешенных или запрещенных для синхронизации в выбранной папке iFolder.

Синхронизация файлов менеджера файлов "thumbs.db" и ".DS_Store" не выполняется.

Чтобы добавить расширение файла в исключающий фильтр, введите его (например *.mpg) и щелкните "Добавить".

Чтобы исключить файл из синхронизации, установите рядом с ним флажок в списке ограничений по типам файлов и щелкните кнопку "Запретить".

Чтобы разрешить синхронизацию определенного типа файлов, установите рядом с ним флажок в списке ограничений по типам файлов и щелкните кнопку "Разрешить".

Чтобы отменить ограничение для определенного типа файла, установите рядом с ним флажок и щелкните кнопку "Удалить".

Значение по умолчанию: "Запрещено", "Разрешить все типы файлов" или "Системная".

Интервал синхронизации  

Установите флажок "Интервал синхронизации", чтобы включить параметр минимального интервала для выбранной папки iFolder, а затем укажите минимальное значение в минутах, которое пользователи могут задать в клиенте iFolder.

Чтобы выключить этот параметр, снимите флажок "Интервал синхронизации". Если данная опция выключена, отображается значение "Без ограничения".

Если данная опция включена, значение минимального интервала синхронизации в минутах позволяет задать минимальное время, затрачиваемое клиентом пользователя на проверку данных iFolder на сервере и в локальных папках iFolder для определения файлов, которые необходимо загрузить или отправить.

Эффективным минимальным интервалом синхронизации всегда является самое высокое значение следующих параметров:

  • Системная политика (по умолчанию имеет значение 5 минут), если не установлена политика пользователя или группы LDAPGroup. Если установлена политика группы LDAPGroup, она переопределяет политику системы. Если установлена пользовательская политика, она переопределяет политику группы LDAPGroup независимо от величины ее значения.

  • Политика локального компьютера или настройка синхронизации с сервером на клиентской рабочей станции

  • Политика iFolder (совокупная).

Значение по умолчанию: "Запрещено", заданное в клиенте iFolder предельное значение.

Совместное использование

 Вкл.
По умолчанию совместное использование iFolder разрешено. Отмените выбор Вкл., чтобы запретить совместное использование выбранной папки iFolder. После применения данной политики совместное использование выбранной папки iFolder будет запрещено даже для ее владельца и администратора.

Аннулировать
Выберите Аннулировать, чтобы удалить всех членов с правами на совместное использование из списка для выбранной папки iFolder.

    ВАЖНО. Обе этих опции отключены, если на уровне системы или пользователя включен и применяется параметр Совместное использование.
