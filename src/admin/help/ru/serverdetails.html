Просмотр подробной информации о сервере

Можно просмотреть и изменить следующую информацию:

Параметр

Описание

Имя (только просмотр)

Имя, назначенное серверу.

Тип

Тип сервера ("Главный" или "Подчиненный"). Чтобы обозначить этот сервер в качестве главного, щелкните кнопку "Установить главным".

Имя DNS

DNS-имя сервера iFolder Enterprise.

Например: 192.168.1.1 или svr1.domain.com.

Внешний URL

Внешний IP-адрес, соответствующий серверу iFolder. Чтобы изменить IP-адрес, измените данный адрес и щелкните "Сохранить" для сохранения внесенных изменений.

Внутренний URL

Внутренний IP-адрес, соответствующий серверу iFolder. Это обеспечивает взаимодействие между серверами в домене iFolder. Внутренний и внешний URL-адреса могут быть одинаковыми. Чтобы изменить данный IP-адрес, введите новый адрес и щелкните "Сохранить" для сохранения внесенных изменений.

Главный URL (отображается только для подчиненного сервера)

IP-адрес, соответствующий серверу iFolder. С помощью этого адреса обеспечивается взаимодействие подчиненного сервера с главным сервером в домене iFolder.

Чтобы изменить IP-адрес, измените данный адрес и щелкните "Сохранить" для сохранения внесенных изменений.

Выберите отчет в раскрывающемся списке, чтобы просмотреть его.

Данная опция недоступна, если на странице "Отчет" не выбран параметр "Включить формирование отчетов".

Просмотр журналов сервера

Можно просмотреть следующую информацию журналов сервера:

Параметр

Описание

Система

Все

Отладка

Отображение операций сервера, которые позволяют службе поддержки Novell выполнить отладку возникших проблем.

Отображение основных операций сервера, которые позволяют службе поддержки Novell устранить возникшие проблемы. Данная опция выбрана по умолчанию.

Предупреждение

Фатальная

Отображение фатальных системных ошибок.

Откл.

Синхронизация электронных персон

Обновление пользователей iFolder в выбранном домене iFolder в соответствии с информацией LDAP через выбранные временные интервалы.

Укажите временной интервал в минутах в поле "Синхронизация электронных персон" и щелкните "Синхронизировать сейчас" для начала синхронизации пользователей iFolder с пользователями LDAP.

Дополнительный интервал для удаления члена

Указание временного интервала, по прошествии которого после удаления пользователя из базы LDAP система iFolder удаляет информацию об этом пользователе с сервера. Данная функция позволяет восстановить данные после случайного удаления пользователя.


Можно изменить следующую связанную с LDAP информацию. Щелкните "Изменить" для изменения значений в новом окне.

Параметр

Описание

Сервер LDAP

Отображение адреса сервера LDAP.

SSL LDAP

Отображение состояния SSL LDAP. Данный параметр также позволяет включить или отключить соединение SSL LDAP.

Прокси-пользователь

Отображение характерного имени прокси-пользователя. Например cn=SimiasProxy,o=novell

Прокси-пользователь iFolder - это электронная персона, с помощью которой осуществляется доступ к серверу LDAP для получения списка пользователей в указанных контейнерах или группах, а также пользователей, заданных в параметрах LDAP iFolder. Прокси-пользователь iFolder создается во время установки iFolder. Эта электронная персона должна иметь право "Чтение" на LDAP каталог.

Пароль прокси-пользователя

 

Пароль используется для аутентификации прокси-пользователя iFolder на сервере LDAP, когда iFolder синхронизирует информацию о пользователях с сервером LDAP.

Контекст LDAP

Отображение списка всех контекстов LDAP. Система iFolder ищет пользователей только в перечисленных контекстах LDAP.


Хранилище данных

Хранилище данных - это хранилище iFolder, которое может охватывать несколько томов (точек монтирования) на отдельном сервере. По умолчанию каждый сервер iFolder имеет хранилище по умолчанию, которое нельзя отключить. С помощью веб-интерфейса можно добавлять и настраивать несколько хранилищ данных, благодаря чему достигается баланс нагрузки для данных iFolder. Когда пользователь отправляет папку iFolder, выполняется поиск хранилища данных с максимальным объемом свободного пространства, после чего данные iFolder сохраняются в этом хранилище данных для балансировки нагрузки. Можно добавить любое количество хранилищ данных. Наличие нескольких хранилищ данных позволяет масштабировать заполненность хранилищ данных при крупном развертывании, что соответствует требованиям уровня корпорации.

Можно просмотреть следующую информацию хранилища данных:

Параметр

Описание

Имя

Отображение уникального имени, заданного для хранилища данных.

Полный путь

Отображение пути к хранилищу данных на смонтированном томе. Путь к данным, указанный при добавлении хранилища данных с помощью веб-интерфейса.

Свободное место

Отображение доступного на томе пространства.

Разрешено

Отображение состояния хранилища данных (включено или нет). Отключить хранилище данных по умолчанию нельзя.

Включение и отключение хранилища данных

Выберите необходимое хранилище данных и щелкните "Запретить" или "Включить". Когда пользователь отправляет iFolder, отключенные хранилища данных всегда пропускаются при проверке максимального объема свободного пространства для сохранения данных iFolder.
