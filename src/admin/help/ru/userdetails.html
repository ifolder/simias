Информация о пользователях

На странице "Информация о пользователе" отображается следующая информация о пользователе для выбранной учетной записи iFolder.

Параметр

Описание

Имя пользователя

Имя пользователя, назначенное учетной записи пользователя, например "jsmith" или "john.smith@example.com".

Полное имя.

Имя и фамилия пользователя в соответствии с его учетной записью.

Тип члена

Тип члена, соответствующий пользователю, вошедшему в систему. Если пользователем является индивидуальный пользователь, в интерфейсе также отображается опция "Группы пользователей". Если пользователем является член группы LDAPGroup, в интерфейсе отображается опция "Члены группы" со списком всех членов этой группы LDAPGroup.

Контексты LDAP

Контекст дерева LDAP используется для персонификации доступа пользователей к iFolder.

Последний вход

Время последнего входа пользователя в систему iFolder.

Групповая квота диска Совокупная квота диска для всей группы.

ПРИМЕЧАНИЕ. Дополнительный администратор не может менять значение групповой квоты диска.

Используемое дисковое пространство Совокупное дисковое пространство, используемое всей группой.

Группы пользователей" (только для индивидуальных пользователей)

Список всех групп, к которым принадлежит выбранный пользователь.

"Члены группы" (только для групп LDAPGroup)

Список всех членов, которые принадлежат к выбранной группе LDAPGroup.

Папки iFolder

На странице "Информация о пользователе" отображаются папки iFolder, находящиеся у пользователя во владении или в совместном использовании. Щелкните вкладку "Все" для отображения списка всех папок iFolder, находящихся во владении или в совместном использовании. Для просмотра папки iFolder, владельцем которой является пользователь, щелкните вкладку "Владелец". На вкладке "Совместное использование" отображается список совместно используемых папок iFolder для текущей учетной записи пользователя.

Включение или отключение папки iFolder

  1. Найдите нужную папку iFolder, а затем установите расположенный рядом с ней флажок.

  2. Щелкните "Включить" для включения папки iFolder.

    Это позволяет пользователю входить в папки iFolder и выполнять их синхронизацию.

  3. Щелкните "Запретить" для запрета доступа к папке iFolder.

    Сеанс пользователя, вошедшего в систему на момент установки данного значения, продолжается до тех пор, пока не будет выполнен выход. Данная политика вступает в силу при следующей попытке пользователя выполнить вход с использованием своей учетной записи. Чтобы блокировка вступила в силу немедленно, необходимо перезапустить сервисы Apache для сервера iFolder. При этом выполняется отключение всех активных сеансов, включая сеансы пользователей.

Удаление папки iFolder

  1. Найдите нужную папку iFolder, а затем установите расположенный рядом с ней флажок.

  2. Щелкните "Удалить".

Создание папки iFolder

  1. На странице "Информация о пользователе" щелкните "Создать".

  2. На открывшейся странице отображаются инструкции, которым необходимо следовать для создания новой папки iFolder.

Политики

Используйте данный интерфейс для управления политиками пользователя для определенной учетной записи пользователя.

ПРИМЕЧАНИЕ. Политики, установленные для группы LDAPGroup, применяются к ее членам.

Изменение политик пользователя

В следующей таблице перечислены политики, которыми можно управлять для любой отдельной учетной записи пользователя iFolder. Измените одно или несколько из следующих значений, а затем щелкните "Сохранить", чтобы новые параметры вступили в силу:

Параметр

Описание

Учетная запись

Отображение информации о том, разрешено ли пользователю в данный момент выполнять вход для синхронизации папок iFolder.

Установите флажок для запрета входа с использованием учетной записи.

Установите флажок для разрешения входа с использованием учетной записи.

Сеанс пользователя, вошедшего в систему на момент установки данного значения, продолжается до тех пор, пока не будет выполнен выход. Данная политика вступает в силу при следующей попытке пользователя выполнить вход с использованием своей учетной записи. Чтобы блокировка вступила в силу немедленно, необходимо перезапустить сервисы Apache для сервера iFolder. При этом выполняется отключение всех активных сеансов, включая сеансы пользователей.

Значение по умолчанию: "Включено"

Число папок iFolder на пользователя

Максимальное разрешенное число папок iFolder, которыми может владеть пользователь. После применения данной политики пользователю будет разрешено владеть только ограниченным числом папок iFolder. Пользователь, превышающий ограничение, получить сообщение о нарушении политики. Если значение ограничения равно нулю, пользователь не может создавать папки iFolder.

Установите флажок рядом с полем Ограничение, чтобы ограничить число папок iFolder, допустимое для одного пользователя, и укажите числовое значение.

Установка политики не влияет на число папок iFolder, которыми пользователь уже владеет. Если число папок iFolder, которыми владеет пользователь, уже превышает данное ограничение, этот пользователь может по-прежнему владеть существующими папками iFolder.

Политика на уровне пользователя переопределяет политику на уровне группы LDAPGroup и системы.

Значение по умолчанию: "Запрещено", значение не установлено

Квота диска

Максимальный общий объем пространства для хранения данных пользователя iFolder во всех папках iFolder, которыми владеет пользователь. Квота может устанавливаться политикой "Системная". Пользовательские политики заменяют собой системные политики.

Лимит
Максимальный общий объем пространства (в Мб) для данных пользователя iFolder во всех папках iFolder, которыми пользователь владеет в рамках данной учетной записи пользователя.

Отмените выбор этой опции, если индивидуальная квота для пользователя отсутствует или для выбранной учетной записи пользователя необходимо использовать квоту на уровне группы LDAPGroup или системную квоту.
Выберите эту опцию для принудительного использования квоты пользователя, а затем укажите общий размер квоты (в Мб) для выбранной учетной записи пользователя.

Используется
Общий объем пространства, используемый в данный момент на сервере для всех папок iFolder, которыми владеет выбранный пользователь.

Доступен
Доступное на сервере свободное пространство.

Действующий
Действительный объем выделенного на сервере пространства.

Если общий объем пространства, занимаемый данными iFolder, приближается к действующей квоте (для системы, группы LDAPGroup, пользователя или папки iFolder), пользователь должен остановить синхронизацию файлов до тех пор, пока с помощью следующих действий не будет получен объем пространства, достаточный для безопасной синхронизации файлов пользователя в папке iFolder:

  • Системная квота, квота группы LDAPGroup, пользовательская квота для владельца папки iFolder и квота iFolder изменены соответствующим образом.

  • Файлы, содержащиеся в любой из папок iFolder, которыми владеет данный пользователь, перемещены в другое расположение, где на них не распространяется действующая квота, или файлы удалены для увеличения объема свободного пространства.

ВАЖНО. Если для пользователя установлено ограничение, которое меньше общего пространства, занимаемого его данными iFolder в текущий момент, синхронизация останавливается до тех пор, пока объем данных не уменьшится и не перестанет превышать ограничение, или значение квоты не будет увеличено и не начнет превышать общий объем пространства, используемый пользователем.

Значение по умолчанию: "Запрещено" или квота не установлена.

Размер файла

 Лимит
Максимальный общий размера файла (в Мб) для всех папок iFolder, которыми пользователь владеет в рамках данной учетной записи пользователя.

Действующий
Действительный размер файла, выделенный для пользователя.

    ВАЖНО. Пользователи не могут выполнить синхронизацию файлов, размер которых нарушает квоту. Если они попытаются сделать это, файл синхронизируется лишь частично, что вызывает повреждение данных.

Значение по умолчанию: "Запрещено" или без ограничения.

Исключенные файлы

Список типов файлов, разрешенных или запрещенных для синхронизации в выбранной учетной записи пользователя.

Синхронизация файлов менеджера файлов "thumbs.db" и ".DS_Store" не выполняется. Хранить данные файлы не требуется, а их синхронизация приведет к повторяющимся ошибкам конфликтов файлов. Если для данного пользователя не установлены индивидуальные ограничения, то файлы "thumbs.db" и ".DS_Store" описываются в данном поле как часть системных ограничений типов файлов. Если для пользователя установлены индивидуальные ограничения типов файлов, то вместо этого отображаются параметры пользователя. Даже если ограничения для файлов "thumbs.db" и ".DS_Store" не отображаются, они действуют всегда и переопределить их нельзя.

Чтобы добавить расширение файла во включающий или исключающий фильтр, введите его (например *.mpg) и щелкните "Добавить".

Чтобы отменить ограничение для определенного типа файла, установите рядом с ним флажок, а затем щелкните "Разрешить".

Значение по умолчанию: ".DS_Store" и "Thumbs.db".

Синхронизация

Минимальный интервал синхронизации в секундах. Для интервала синхронизации можно задать значение от 5 до 999 включительно. Если значение не задано, параметр имеет значение 5.

Снимите флажок, чтобы не использовать интервал синхронизации или принять для учетной записи пользователя системное значение. Если данное значение не задано для системных или пользовательских политик, то отображается значение "Без ограничения".

Интервал
Установите этот флажок, чтобы использовать минимальный интервал синхронизации, а затем укажите значение (в минутах). Например, можно использовать значение, равное 10 минутам.

Действующий
Текущий интервал синхронизации. Например, если установленный пользователем интервал синхронизации меньше (синхронизация выполняется чаще), чем минимальное системное значение, применяется системное значение.

Действующим минимальным интервалом синхронизации всегда является самое высокое значение следующих параметров:

  • Системная политика (по умолчанию имеет нулевое значение), если не установлена пользовательская политика или политика группы LDAPGroup. Если установлена пользовательская политика или политика группы LDAPGroup, она переопределяет системную политику независимо от величины значения.

  • Политика локального компьютера или настройка синхронизации с сервером на клиентской рабочей станции.

  • Политика iFolder (совокупная).

Значение по умолчанию: "Запрещено" или системная политика.

Шифрование

Политика шифрования для выбранной учетной записи пользователя. Квота может устанавливаться системной политикой ("Системная"). "Системная" политика заменяет собой пользовательские политики.

Вкл.
Выберите "Вкл.", чтобы включить функцию шифрования для выбранного пользователя. Это позволит пользователю установить политику шифрования для своих папок iFolder.

Отмените выбор значения "Вкл.", чтобы выключить функцию шифрования для выбранного пользователя.

Принудительно
Выберите "Принудительно", чтобы включить функцию шифрования для выбранного пользователя. Если установлено значение "Принудительно", пользователь не может изменить параметры шифрования для своих папок iFolder.

ВАЖНО. Данная опция доступна только в том случае, если включена системная политика шифрования.

Совместное использование

 Вкл.
По умолчанию совместное использование iFolder разрешено. Отмените выбор значения "Вкл." и щелкните "Сохранить", чтобы запретить совместное использование для выбранного пользователя. После применения данной политики выбранный пользователь не сможет совместно использовать свои папки iFolder с другими пользователями. Однако можно изменить политику на уровне папки iFolder.

Enforce (Принудительно использовать)
Выберите Enforce (Принудительно использовать) для принудительного применения политики к выбранному пользователю. После применения данной политики выбранный пользователь не сможет совместно использовать свои папки iFolder.

Аннулировать
Выберите Аннулировать, чтобы удалить членов с правами на совместное использование для всех папок iFolder, принадлежащих выбранному пользователю.
