Описание

Порядок отображения полных имен

Позволяет задать порядок, в котором отображается полное имя пользователя. Опция (Имя, фамилия) предписывает выводить сначала имя, затем — фамилию. Чтобы отображать сначала фамилию, а затем имя, выберите опцию (Фамилия, имя).

Изменения вступят в силу только после запланированной или ручной синхронизации LDAP. Для того чтобы провести синхронизацию LDAP вручную, выполните указанные ниже действия.

  1. В консоли веб-администрирования откройте вкладку Серверы, выберите сервер и откройте страницу с подробной информацией о сервере.

  2. В разделе Подробная информация LDAP щелкните кнопку Синхронизировать сейчас.

Способ управления групповой квотой

Позволяет выбрать способ управления совокупной квотой диска, определенной для групп.

Опция Консоль администратора позволяет администраторам явным образом управлять квотой диска для групп и их членов. Выбор этой опции ограничивает квоту диска, назначенную пользователям, совокупным объемом используемого группой места на диске. Опция Консоль администратора выбрана по умолчанию.

Опция Подсистема синхронизации позволяет управлять совокупной квотой диска для групп в серверном компоненте с помощью подсистемы синхронизации. Подсистема синхронизации проверяет, чтобы квота диска для всех пользователей группы не превышала совокупную квоту диска группы.

 

Выделенные группы Позволяет выделять группы в независимые объекты и изолировать при совместном использовании папок iFolder членов одной группы от доступа к ним членов другой группы. Установите флажок Создать выделенные группы, чтобы разрешить совместное использование папок iFolder только в пределах групп.

Примечание. Группа LDAPGroup не может быть ни дополнительным, ни основным администратором.

Состояние изменения доступа

Можно перемещать пользователей на другие серверы. Данный интерфейс позволяет просматривать состояние изменения доступа. Щелкните "Состояние изменения доступа", чтобы просмотреть для каждого пользователя информацию о состоянии изменения доступа.

Администраторы iFolder

Администратор iFolder - это основной администратор сервера iFolder. Администратор iFolder назначается владельцем папок iFolder с нарушенными ассоциациями до тех пор, пока они не будут переназначены другому владельцу или удалены. Первоначально администратор iFolder задается при настройке конфигурации сервера iFolder в YaST.

Для выполнения задач управления администратор iFolder должен обладать персонифицированным доступом. Система iFolder отслеживает данного пользователя по идентификатору GUID объекта LDAP, поэтому он может относиться к любому контейнеру или группе LDAP в дереве (включая не входящие в список "Поиск характерных имен"). Поскольку для идентификации используется идентификатор GUID, а не характерное имя, перемещение пользователя может отслеживаться в любой части дерева.

Права администратора iFolder можно назначить другим пользователям, чтобы они также смогли управлять сервисами iFolder для выбранного сервера. Правами, эквивалентными правам администратора iFolder, могут обладать только пользователи, которые относятся к одному из характерных имен, указанных в списке "Поиск характерных имен" LDAP.

Если права администратора iFolder назначаются другим пользователям, то действия этих пользователей регулируются реестром и списком "Поиск характерных имен". Если пользователь удален, перемещен на характерное имя, не входящее в список "Поиск характерных имен", или характерное имя пользователя удалено из списка "Поиск характерных имен", то этот пользователь удаляется из реестра и лишается прав администратора iFolder.

Администратор iFolder или основной администратор создает дополнительного администратора и назначает ему группы.

ВАЖНО. Назначение прав администратора на использование группы LDAPGroup невозможно.

На странице "Система" есть вкладки с параметрами основного и дополнительного администраторов. Ниже перечислены параметры, общие для обеих этих вкладок.

Параметр

Основной администратор

Чтобы просмотреть или изменить параметры основного администратора, откройте вкладку Основной администратор и щелкните ссылку администратора. В результате откроется страница с подробной информацией этого пользователя. На странице "Информация о пользователе" отображаются папки iFolder, находящиеся у администратора во владении или в совместном использовании. Щелкните вкладку "Все" для отображения списка всех папок iFolder, находящихся во владении или в совместном использовании. Для просмотра папки iFolder, владельцем которой является пользователь, щелкните вкладку "Владелец". На вкладке "Совместное использование" отображается список совместно используемых папок iFolder для данной учетной записи пользователя. На странице информации о пользователе можно изменить параметры политики для выбранного администратора.

Добавление и удаление администратора iFolder


  1. Найдите нужного администратора, а затем установите расположенный рядом с ним флажок.

    • Щелкните "Удалить" для удаления выбранного администратора.

      Если удалить права администратора iFolder для имени пользователя, с помощью которого Вы вошли на сервер, Вы будете немедленно отключены. Чтобы продолжить управление сервером iFolder, Вам придется войти на него, используя другое имя пользователя с правами администратора iFolder.

      ВАЖНО. Удалить администратора, который был задан при настройке сервера Simias, нельзя.

    • Щелкните "Добавить", чтобы открыть список пользователей iFolder.

      Найдите пользователя, которого необходимо добавить в качестве администратора, затем установите рядом с ним флажок и щелкните "Добавить".

      Права администратора iFolder можно назначать нескольким пользователям.

Политики

В следующей таблице перечислены политики, которыми можно управлять для любой отдельной системы iFolder. Щелкните "Сохранить", чтобы изменения вступили в силу:

Параметр

Описание

Число папок iFolder на пользователя

Максимальное разрешенное число папок iFolder для одного пользователя. После применения данной политики каждому пользователю будет разрешено владеть только ограниченным числом папок iFolder. Пользователи, превышающие это ограничение, будут получать сообщения о нарушении политики. Если значение ограничения равно нулю, пользователи не могут создавать папки iFolder.

Установка политики не влияет на число папок iFolder, которыми пользователь уже владеет. Если число папок iFolder, которыми владеет пользователь, уже превышает данное ограничение, этот пользователь может по-прежнему владеть существующими папками iFolder.

Квота диска

Максимальный объем пространства, который может использовать система iFolder. Системные параметры заменяют собой пользовательские политики.

Снимите этот флажок, чтобы отключить системную квоту.

Установите этот флажок, чтобы включить системную квоту, а затем укажите общую квоту пространства (в Мб) для учетной записи пользователя.

Если включена системная квота, которая меньше общего пространства, занимаемого данными iFolder в текущий момент, синхронизация останавливается до тех пор, пока объем данных не уменьшится и не перестанет превышать ограничение, или значение квоты не будет увеличено и не начнет превышать общий объем пространства, используемый пользователем.

Включение или изменение системной квоты не влияет на существующие индивидуальные квоты пользователей. Любая существующая квота пользователя всегда переопределяет системную квоту независимо от величины квоты пользователя.

Значение по умолчанию: "Запрещено", "Нет ограничения".

Размер файла

Максимальный размер файла, при котором возможна синхронизация.

Если указана квота, то действующее ограничение максимального размера файла приравнивается к ее значению.

Снимите флажок, чтобы отключить политику "Ограничение максимального размера файла". Если данная политика отключена, отображается значение "(Нет ограничения)".

Установите данный флажок, чтобы включить политику "Ограничение максимального размера файла", а затем укажите максимальный размер файла в Мб.

Если указана квота, то ограничение максимального размера файла по умолчанию приравнивается к ее значению.

Выясните следующие аспекты, чтобы определить подходящее ограничение размера файла для папок iFolder в Вашей среде:

  • Предполагаемое использование

  • Частота изменения наиболее крупных файлов

  • Используемый в приложениях способ сохранения изменений в крупных файлах (сохранение всего файла или отдельных блоков)

  • Частота синхронизации файлов каждым из членов

  • Количество пользователей, осуществляющих совместное использование iFolder

  • Способ доступа пользователей к iFolder: посредством локальной сети, ГС или Интернет-соединения

  • Средняя и пиковая пропускная способность

Даже если в качестве ограничения размера файла установлено очень большое значение, и отсутствует квота, ограничивающая размеры файлов, действительное значение определяется файловой системой на компьютере пользователя. Например, размер томов FAT32 не может превышать значения 4 Гб минус 1 байт.

Значение по умолчанию: "Запрещено", "Нет ограничения".

Исключенные файлы

Определение ограничения типов файлов, которые синхронизируются с помощью исключающих фильтров.

Введите расширение файла, а затем щелкните "Добавить", чтобы добавить его в список.

Чтобы исключить файл из синхронизации, установите рядом с ним флажок в списке ограничений по типам файлов и щелкните кнопку "Запретить".

Чтобы разрешить синхронизацию определенного типа файлов, установите рядом с ним флажок в списке ограничений по типам файлов и щелкните кнопку "Разрешить".

Чтобы отменить ограничение для определенного типа файла, установите рядом с ним флажок, а затем щелкните кнопку "Удалить".

Синхронизация

Минимальный интервал синхронизации в минутах.

Если данная опция включена, она задает минимальный интервал синхронизации данных iFolder для каждой учетной записи пользователя. Чем больше используемое значение, тем больше ограничение.

Если данная опция не включена, используется значение по умолчанию 5 минут.

Шифрование

Определение политики шифрования для системы iFolder. Системные параметры заменяют собой пользовательские политики.

Вкл.
Выберите Вкл., чтобы включить функцию шифрования для системы iFolder. Это позволит пользователю установить политику шифрования для своих папок iFolder.

Отмените выбор значения Вкл., чтобы выключить функцию шифрования для всех пользователей.

Принудительно
Выберите "Принудительно", чтобы включить функцию шифрования для всех пользователей. Если для политики шифрования установлено значение "Принудительно", пользователь не может изменить параметры шифрования для своих папок iFolder.

Совместное использование

 Вкл.
По умолчанию совместное использование iFolder разрешено. Отмените выбор значения Вкл., чтобы выключить функцию совместного использования в системе iFolder. После применения данной политики пользователи этой системы iFolder не смогут совместно использовать свои папки iFolder с другими пользователями. Однако можно изменить параметры политики на уровне пользователя или группы LDAPGroup.

Enforce (Принудительно использовать)
Выберите Enforce (Принудительно использовать) для принудительного применения политики совместного использования, установленной на уровне системы. С помощью функции принудительного использования можно как разрешить, так и запретить совместное использование. В случае принудительного запрещения совместного использования параметры совместного использования на уровне папки iFolder и пользователя автоматически отключаются, а их изменение становится невозможным. Однако можно установить для данной политики опцию "Аннулировать".

    ВАЖНО. Для включения данной опции необходимо установить для опции "Совместное использование" значение "Вкл".


Аннулировать
Выберите Аннулировать, чтобы удалить членов с правами на совместное использование для всех папок iFolder в данной системе iFolder.

Дополнительный администратор

На вкладке "Дополнительный администратор" можно добавлять, изменять и удалять параметры дополнительного администратора.

Чтобы добавить дополнительного администратора, выполните указанные ниже действия.

  1. Откройте вкладку Дополнительный администратор и щелкните кнопку Добавить, чтобы открыть список пользователей iFolder.
  2. Выберите пользователя, которого хотите назначить дополнительным администратором, и щелкните Далее.
  3. Чтобы назначить дополнительному администратору группу, выберите соответствующий вариант из списка Выбор группы. Если для выбранной группы установлена предельная совокупная квота диска, в поле Установите предельную совокупную квоту диска для всей группы подставлено соответствующее значение. В противном случае в этом поле ничего нет.
  4. Установите правила политики для дополнительного администратора.

  2. Назначив дополнительному администратору группу, щелкните ОК, чтобы вернуться на страницу "Система", или Повторить, чтобы назначить ему и другие группы. 

Чтобы изменить параметры дополнительного администратора, выполните указанные ниже действия.

  1. Откройте вкладку Дополнительный с подробной информацией о дополнительном администраторе.

  2. Выберите дополнительного администратора и щелкните Изменить. В результате откроется список групп, за которыми он наблюдает.

  3. Выберите группу и щелкните Изменить. В результате откроется список прав дополнительного администратора на эту группу. Внесите в права нужные изменения и щелкните Сохранить, чтобы сохранить изменения.
     

Чтобы удалить дополнительного администратора, выполните указанные ниже действия.

  1. Откройте вкладку Дополнительный с подробной информацией о дополнительном администраторе.
  2. Выберите дополнительного администратора и щелкните Удалить. В результате откроется список групп, за которыми он наблюдает.
  3. Выберите все группы и щелкните Удалить. Удаление всех групп из списка групп, принадлежащих дополнительному администратору, приведет к удалению и самого администратора.
