管理 iFolder 细节

使用此界面可管理所选 iFolder 的所有权、策略和成员资格。

注意：二级管理员可以根据二级管理员在选定 iFolder 上的权限管理选定 iFolder 的策略。

拥有者

所选 iFolder 的拥有者的用户名。拥有者可以是单个用户，也可以是 LDAP 组。如果 iFolder 的拥有者是 LDAP 组，则 LDAP 组的所有成员都将具有相同的访问权限。对于孤立的 iFolder，在可以重指派或删除 iFolder 之前，iFolder Admin 用户是其拥有者。

单击该拥有者的用户名链接可查看其用户帐户的细节。

iFolder 拥有者具有对该 iFolder 的“完全控制”权限。该拥有者可管理用户的成员资格和访问权限，并且可去除任何成员的“完全控制”权限。

成员

您可以查看成员的名称、类型和指派给他们的访问权限。允许您添加或删除拥有者、指派所有权和设置对所选成员的访问权限。

添加成员

  1. 在“iFolder 细节”页上，单击“添加”。

  2. 搜索要添加为成员的用户或 LDAP 组，选中用户名旁边的复选框，然后单击“添加”。

    默认情况下，该用户或 LDAP 组对 iFolder 具有只读访问权限。

  3. （可选）选中“用户”复选框，然后选择“权限”并将访问权限指定为“Admin”、“读/写”或“只读”，然后单击“设置”。

    等待页面刷新。用户的图标应该反映新的访问权限。

了解 iFolder 访问权限

iFolder 客户程序支持拥有者和三个 iFolder 成员访问级别：

注意：LDAP 组的成员将继承为该 LDAP 组设置的访问权限。

拥有者
一个 iFolder 的拥有者只能是一个用户或一个 LDAP 组。通常是创建该 iFolder 的用户或 LDAP 组的任意成员。如果拥有者是一个 LDAP 组，则此组的所有成员都将具有该 iFolder 的所有权。作为拥有者的用户或 LDAP 组的成员可使用 iFolder 客户程序将所有权转让给该 iFolder 的其他成员。iFolder Admin 用户可将所有权转让给该 iFolder 服务器上的任何用户或 LDAP 组。

iFolder 的拥有者具有“完全控制”权限。此用户或 LDAP 组对该 iFolder 具有读/写权限，可以管理成员用户和 LDAP 组的成员资格和访问权限，还可以去除任何成员的完全控制权限。如果使用的是企业服务器，将根据拥有者在企业服务器上的用户磁盘定额计算该拥有者的 iFolder 所使用的磁盘空间。

如果从 iFolder 系统中删除某一用户或 LDAP 组，则由该用户或 LDAP 组所有的 iFolder 将被孤立。孤立的 iFolder 的所有权被临时指派给 iFolder Admin 用户。当 iFolder Admin 用户决定是应删除孤立的 iFolder 还是应将其分配给新的拥有者时，其他用户和 LDAP 组成员将继续拥有成员资格并可进行同步。

完全控制
共享 iFolder 的成员，具有“完全控制”访问权限。具有完全控制权限的用户或 LDAP 组对 iFolder 具有读/写权限，并管理除拥有者以外的所有用户的成员资格和访问权限。

读/写
共享 iFolder 的成员，对 iFolder 中的目录和文件具有“读/写”访问权限。

只读
共享 iFolder 的成员，对 iFolder 中的目录和文件具有“只读”访问权限。该成员可将 iFolder 文件复制到另一个位置并在 iFolder 外修改它。

为成员设置 iFolder 访问权限

  1. 在“iFolder 细节”页上，选中要管理其访问权限的成员旁边的复选框。

  2. 选择“权限”下拉菜单，然后选择“Admin”、“读/写”或“只读”权限。

    等待页面刷新。用户的图标应该反映新的访问权限。

去除成员

  1. 在“iFolder 细节”页上，选中成员用户名旁边的复选框。
  2. 单击“删除”。

用户的数据本地拷贝仍保留在该用户的计算机上，但该用户将不能再访问 iFolder 数据的服务器拷贝。

更改拥有者

更改 iFolder 的拥有者时，现有拥有者将成为该 iFolder 的一个成员，并被指派“读/写”权限。对于孤立的 iFolder，iFolder Admin 将成为拥有者，直到该 iFolder 被重新指派给新的拥有者或被删除。如果新的拥有者当前不是 iFolder 的成员，系统将向该用户的 iFolder 帐户发送订购通知。

  1. 在“iFolder 细节”页上，搜索要指派为该 iFolder 的新拥有者的用户或 LDAP 组。

  2. 选中该用户名称旁边的复选框，然后单击所有者。

策略

下表列出了可为任意给定 iFolder 进行管理的策略。修改以下的一个或多个值，然后单击“保存”以应用新的设置：

参数

说明

禁用同步

选择此选项可关闭同步（通常是临时性的）。

取消选择此选项可同步 iFolder 中的数据。

默认值：已启用，是

磁盘定额

选中“限制”复选框，然后指定所选 iFolder 的最大大小（以 MB 计）。

如果启用了系统范围的 iFolder 定额或 LDAP 组级定额，且用户定额低于或高于 LDAP 组定额或系统定额，则用户的帐户定额将覆盖这些定额。

默认值：已禁用，无限制

已用空间

（仅供查看）

报告 iFolder 数据当前占用的空间。

可用空间

（仅供查看）

报告服务器上 iFolder 数据的可用空间。

有效空间

（仅供查看）

报告服务器上 iFolder 数据可用的有效空间。

文件大小

 限制
指定在 iFolder 用户的选定用户帐户所拥有的所有 iFolder 中，允许该用户使用的最大文件总大小（以 MB 计）。

有效
分配给用户的有效文件大小。

    重要说明：用户不能成功同步大小超出定额的文件。如果用户尝试这样做，则只有部分文件被同步，这将导致数据损坏。

排除的文件

为所选 iFolder 指定要包括到同步中的文件类型列表或要从同步中排除的文件类型列表。

名为 thumbs.db 和 .DS_Store 的文件管理器文件将永远不会被同步。

要将文件扩展名添加到排除过滤器，请先键入扩展名（如 *.mpg），然后单击“添加”以应用过滤器。

要从受限的文件类型中排除文件类型，请先选中“文件名”旁边的复选框，然后选择文件类型并单击“删除”。

默认值：已禁用，允许所有文件类型或系统级设置

同步间隔

选中“同步间隔”复选框为所选 iFolder 启用最小间隔设置，然后指定用户可以在其 iFolder 客户程序上进行设置的最小间隔值（分钟）。

要禁用此设置，请取消选中“同步间隔”复选框。如果禁用了此选项，则报告的值不受限制。

如果已启用此选项，则最小同步间隔指定用户的客户程序可以检查服务器上的 iFolder 数据和本地 iFolder 以标识需要下载或上载文件的最小间隔（分钟）。

有效最小同步间隔始终是下列设置中的最大值：

  • 系统策略（默认值为 5 分钟），除非设置了用户策略或 LDAP 组策略。如果设置了 LDAP 组策略，LDAP 组策略将覆盖系统策略。如果设置了用户策略，则无论该用户策略的值是大于还是小于 LDAP 组策略和系统策略中的值，该用户策略都将覆盖这些策略。

  • 本地计算机策略，或与服务器同步的客户机上的设置

  • iFolder（集合）策略

默认值：禁用、iFolder 客户程序设置限制

共享

 启用
默认情况下，将启用 iFolder 共享。取消选中启用以禁止共享选定 iFolder。应用此策略后，Admin 或 iFolder 的所有者将无法共享选定 iFolder。

撤消
选择撤消可从选定 iFolder 的共享成员列表中去除所有成员。

    重要说明：如果您在系统或用户级别启用并强制使用共享选项，则将同时禁用这两个选项。
