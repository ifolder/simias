用户细节

使用此界面查看用户细节并管理用户策略。

注意：术语 iFolder 用户 既指单个用户，也指 LDAP 组。

iFolder
策略

用户细节

“用户细节”页将显示所选用户的 iFolder 帐户的以下用户细节。

参数

说明

用户名

指派给用户帐户的用户名，例如 jsmith 或 john.smith@example.com。

Full name

用户帐户的名和姓。

成员类型

显示当前登录的用户的成员类型。如果该用户是单个用户，则界面上还会显示出一个“用户组”选项。如果该用户是 LDAP 组的成员，将在界面上的“组成员”选项下列出该组的所有成员。

LDAP 环境

LDAP 树环境用于向 iFolder 提供用户。

上次登录

用户上次登录到 iFolder 系统的时间。

组磁盘定额 代表组的累积磁盘定额。

注意：二级管理员无法修改组磁盘定额。

已用磁盘空间： 表示组已使用的累积磁盘空间。

用户组（仅对单个用户可用）

列出选定用户所属的所有组。

组成员（仅适用于 LDAP 组）

列出属于选定 LDAP 组的所有成员。

iFolder

“用户细节”页显示用户拥有或共享的 iFolder。单击“所有”选项卡可列出用户拥有并共享的所有 iFolder。若要查看用户所拥有的 iFolder，请单击“拥有”选项卡。“共享”选项卡列出当前用户帐户的所有共享的 iFolder。

启用或禁用 iFolder

  1. 查找要管理的 iFolder，然后选中该 iFolder 旁边的复选框。

  2. 单击“启用”可启用该 iFolder。

    这将允许用户登录并同步 iFolder。

  3. 单击“禁用”可禁用该 iFolder。

    如果用户在您进行此更改时已经登录，则在该用户注销之前，该用户的会话将继续。此策略将在用户下次试图登录该帐户时生效。要使锁定立即生效，则必须重启动 iFolder 服务器的 Apache 服务，该服务器将中断所有的活动会话，包括用户的会话。

删除 iFolder

  1. 找到您要删除的 iFolder，然后选中该 iFolder 旁边的复选框。

  2. 单击“删除”。

创建 iFolder

  1. 在“用户细节”页上，单击“新建”。

  2. 在打开的新页上，遵循说明创建一个新的 iFolder。

策略

使用此界面管理指定的用户帐户的用户策略。

注意：对 LDAP 组设置的策略适用于其成员用户。

修改用户策略

下表列出了您可以为任意给定的 iFolder 用户帐户管理的策略。修改以下的一个或多个值，然后单击“保存”以应用新的设置：

参数

说明

帐户

指定当前是否允许用户登录以同步 iFolder。

选中该复选框以禁用登录帐户。

取消选中该复选框以启用登录帐户。

如果用户在您进行此更改时已经登录，则在该用户注销之前，该用户的会话将继续。此策略将在用户下次试图登录该帐户时生效。要使锁定立即生效，则必须重启动 iFolder 服务器的 Apache 服务，该服务器将中断所有的活动会话，包括用户的会话。

默认值：启用

每个用户的 iFolder 数

指定每个用户可拥有的 iFolder 的最大数目。应用此策略后，将限制用户只能拥有特定数量的 iFolder。超出各自的用法限制的用户将接收到有关违反策略的错误讯息。如果该限制为零，则用户无法创建任何 iFolder。

选中限制旁边的复选框以启用每个用户的 iFolder 限制，并在该字段中指定数目。

该策略设置不会影响用户已拥有的 iFolder 的数目。如果用户已拥有的 iFolder 数超过您设置的限制，则用户仍然拥有这些 iFolder。

用户级别的策略将覆盖 LDAP 组级别和系统级别的策略。

默认值：禁用且未设置值

磁盘定额

在用户拥有的所有 iFolder 中，指定允许用户的 iFolder 数据使用的最大总空间。该定额可以由系统级策略强用。用户策略优先于系统级策略。

限制
指定在 iFolder 用户的选定用户帐户所拥有的所有 iFolder 中，允许该用户的 iFolder 数据所使用的最大总大小（以 MB 计）。

如果没有单个用户定额，或者要为选定用户帐户接受 LDAP 组级别定额或系统范围的定额，请取消选中此选项。
选中此选项以实施用户定额，然后为所选的用户帐户指定总空间定额（以 MB 计）。

已使用
指定此选定用户拥有的所有 iFolder 当前在服务器上占用的总空间。

可用空间
服务器上可用的空间

有效
服务器上已分配的有效空间

如果 iFolder 数据占用的总空间接近有效定额（系统、LDAP 组、用户或 iFolder），则用户应停止同步文件，直到下列一个或多个任务生成足够的空间，以便在用户的文件所在的 iFolder 内安全地同步这些文件：

  • 根据需要修改 iFolder 拥有者的系统范围定额、LDAP 组定额和用户定额，以及 iFolder 定额。

  • 从用户拥有的任何 iFolder 中将文件移动到另一个位置（这些文件在此位置将不再影响有效的定额），或删除这些文件以清理空间。

重要说明：如果您启用的用户空间限制小于用户的 iFolder 数据的当前总空间，则用户数据将停止同步，直到数据减少到低于此限制，或直到定额增加到一个大于用户已占用总空间的值。

默认值：已禁用并且未设置任何定额

文件大小

 限制
指定在 iFolder 用户的选定用户帐户所拥有的所有 iFolder 中，允许该用户使用的最大文件总大小（以 MB 计）。

有效
分配给用户的有效文件大小。

    重要说明：用户不能成功同步大小超出定额的文件。如果用户尝试这样做，则只有部分文件被同步，这将导致数据损坏。

默认值：已禁用并且没有限制

排除文件

为所选用户帐户指定一个要在同步中包含或排除的文件类型列表。

名为 thumbs.db 和 .DS_Store 的文件管理器文件将永远不会被同步。无需保存这些文件，而且同步这些文件将导致重复的文件冲突错误。如果您没有为此用户设置任何单独的限制，则此字段将 thumbs.db 和.DS_Store 作为系统级文件类型限制的一部分报告。而在您为该用户设置了单独的文件类型限制后，将显示该用户的设置。即使未显示 thumbs.db 和 DS_Store 限制，它们也将始终适用；您无法覆盖它们。

要将一个文件扩展名添加到一个包含或排除过滤器，请键入该扩展名（例如 *.mpg），然后单击“添加”以应用该过滤器。

要排除同步某个文件类型，请选中限制文件类型列表中该文件类型旁的复选框，然后单击“拒绝”。

要允许同步某个文件类型，请选中限制文件类型列表中该文件类型旁的复选框，然后单击“允许”。

要从限制文件类型列表中删除某个文件类型，请选中该文件类型旁边的复选框，然后单击“删除”。

默认值：.DS_Store 和 Thumbs.db

同步

指定最小同步间隔（以秒计）。可以将同步间隔设置为 5 到 999（包括 999）之间的某一个值。如果未设置任何值，则该值为 5。

取消选中该复选框可不设置任何同步间隔或接受用户帐户的系统级设置。如果没有为系统级策略或用户策略设置任何值，则报告的值为“无限制”。

间隔
选中该复选框以启用最小同步间隔，然后指定最小间隔（以分钟计）。例如，一个切实可行的值为 10 分钟。

有效
指定当前同步间隔。例如，如果用户设置了一个小于系统最小值的同步间隔（通常情况下），则该系统设置适用。

有效的最小同步间隔始终是以下设置中的最长的值：

  • 系统策略（默认值为零），除非设置了 LDAP 组策略或用户策略。如果设置了用户策略，则无论用户间隔的值是短于还是长于系统策略和 LDAP 组策略中的值，该用户策略都将覆盖这些策略。

  • 本地计算机策略或客户机上的设置与服务器同步。

  • iFolder（集合）策略。

默认值：已禁用，或系统级策略

加密

为所选的用户帐户指定加密策略。该定额可以由系统策略（系统级）强用。系统级策略优先于用户策略。

启用
选中“启用”以为所选的用户启用加密功能。这将允许用户为他/她的 iFolder 设置加密策略。

取消选择“启用”可为所选的用户禁用加密功能。

强制
选中“已实施”以为选中的用户启用加密功能。当加密功能设置为“已实施”时，用户无法为他/她的 iFolder 更改加密设置。

重要说明：仅当系统级加密策略设置为“启用”时，才可启用此选项。

共享

 启用
默认情况下，将启用 iFolder 共享。取消选中“启用”并单击“保存”可禁用选定用户的共享。应用此策略后，选定用户将不能与他人共享自己的 iFolder。但是，您仍然可以在 iFolder 级别更改策略。

强制
选择强制将强制应用为选定用户设置的策略。应用此策略后，用户将不能共享自己的 iFolder。

撤消
选择撤消将去除属于选定用户的所有 iFolder 的共享成员。
