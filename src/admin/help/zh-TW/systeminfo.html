完整名稱顯示順序

讓您設定顯示使用者全名的順序。選取「(名字，姓氏)」選項，以便先顯示名字再顯示姓氏。或者，若要先顯示姓氏再顯示名字，請選取「(姓氏，名字)」選項。

若要讓變更生效，必須等到排程的 LDAP 同步作業執行，或手動執行 LDAP 同步作業。執行手動 LDAP 同步作業：

  1. 在 Web 管理員主控台中，按一下「伺服器」索引標籤、選取伺服器，然後移至「伺服器詳細資料」頁面。

  2. 在「LDAP 細節」區段中，按一下「立即同步化」按鈕。

使用下列項目管理群組配額

讓您定義如何管理為群組設定的彙總磁碟配額。

選取「管理員主控台」選項可讓管理員明確地管理群組的磁碟配額與群組的成員。當您選取此選項時，會限制指定給使用者的磁碟配額，這樣使用者可使用的磁碟空間就不會超過群組的彙總磁碟使用空間。根據預設，會選取「管理員主控台」選項。

選取「同步引擎」選項可讓同步引擎在後端管理群組上的彙總磁碟配額。同步引擎可確保群組中所有使用者的磁碟配額不會超過群組的彙總磁碟配額。

 

隔離的群組 讓您將群組隔離為獨立的實體，並確保屬於一個群組之成員的共享 iFolder 不會被另一個群組的成員存取。選取「建立隔離的群組」核取方塊可限制只有相同群組的成員才能共享 iFolder。

注意：LDAPGroup 不能是次要或主要管理員。

重新供應狀態

您可以在不同伺服器間移動使用者。有了這個介面，您可以檢視重新供應狀態。按一下「重新供應狀態」即可檢視每個使用者的重新供應狀態。

iFolder 管理員

「iFolder 管理員」使用者是 iFolder 企業伺服器的主要管理員。每當有 iFolder 殘留下來時，擁有權便會轉讓給「iFolder 管理員」使用者，以便重新指定給其他使用者或刪除。您一開始會在於 YaST 內設定 iFolder 企業伺服器的組態時，指定「iFolder 管理員」使用者。

「iFolder 管理員」使用者必須處於供應狀態，如此一來「iFolder 管理員」才能執行管理工作。iFolder 會藉由 LDAP 物件 GUID 追蹤這個使用者，使其屬於任何 LDAP 容器或網路樹內的群組，甚至是在其無法識別為「搜尋 DN」的情形下。使用者在網路樹內任何地方的移動都可以追蹤得到，因為可以從 GUID 得知，而不是藉由使用者 DN 得知。

「iFolder 管理員」權限可以指定給其他使用者，使這些使用者也可以管理選定伺服器的 iFolder 服務。只有屬於「LDAP 搜尋 DN」內指定之 DN 一員的使用者才具有擔任「iFolder 管理員」使用者的資格。

如果您將「iFolder 管理員」權限指定給其他使用者，則這些使用者會受到名冊和「搜尋 DN」關係的控制。如果您刪除使用者、從「搜尋 DN」清單移除使用者的 DN，或將使用者移動至不位於「搜尋 DN」的 DN，則使用者會從名冊中刪除，並會卸除其「iFolder 管理員」權限。

iFolder 管理員使用者或主要管理員可建立次要管理員，並指定群組給次要管理員。

重要：您不能指定管理員使用者權限給 LDAPGroup。

主要管理員

若要檢視或編輯主要管理員詳細資料，請按一下「主要管理員」索引標籤，然後按一下管理員使用者連結以開啟「使用者詳細資料」頁面。「使用者詳細資料」頁會顯示 Admin 使用者擁有或共用的 iFolder。按一下「全部」索引標籤以列出所有的 iFolder，包含擁有或共用的 iFolder。若要檢視使用者擁有的 iFolder，請按一下「擁有」索引標籤。「共用」索引標籤會列出這個特定使用者帳戶所有共用的iFolder。您也可以使用「使用者詳細資料」頁變更所選「管理員」使用者的規則設定。

加入及刪除 iFolder 管理員


  1. 找出您要管理的 Admin 使用者，然後選取該使用者旁的核取方塊。

    • 按一下「刪除」以刪除選定的 Admin 使用者。

      如果您從用於登入伺服器的使用者名稱刪除「iFolder 管理員」權限，便會立即中斷與伺服器的連接。您必須以另一個具有「iFolder 管理員」權限的使用者名稱登入 iFolder 伺服器，才能繼續管理伺服器。

      重要：您不能刪除在 Simias 伺服器安裝期間設定的「管理員」使用者。

    • 按一下「新增」以開啟 iFolder 使用者清單。

      找出您要加入為「管理員」的使用者，然後選取該使用者旁的核取方塊，並按一下「新增」。

      您可以指定「iFolder 管理員」權限給多個使用者。

規則

下表列出您可以管理任何指定「iFolder 系統」的系統規則。按一下「儲存」以套用修改：

參數

描述

每個使用者的 iFolder

指定每個使用者能擁有的 iFolder 數量上限。套用此規則後，每個使用者將只能擁有固定數目的 iFolder。超出使用上限的使用者將會收到違反規則相關的錯誤訊息。如果限制為零，使用者無法建立任何 iFolder。

規則設定不影響使用者已擁有的 iFolder 數目。如果使用者擁有的 iFolder 數量超過設定的限制，使用者仍可擁有其 iFolder。

磁碟配額

指定 iFolder 系統可以使用的最大總空間。系統範圍設定會取代使用者規則。

取消選取核取方塊，以停用系統範圍的配額。

選取核取方塊，以啟用系統範圍的配額，然後再為使用者的帳戶指定總配額空間 (以 MB 為單位)。

如果您啟用了系統範圍的配額，而該配額小於使用者目前 iFolder 資料的總空間，則使用者的配額資料會停止同步，直到增加的資料小於限制，或配額增加到大於使用者消耗之總空間的值為止。

啟用或修改系統範圍的配額不會影響到現有的個別使用者配額。現有的使用者配額永遠都會覆寫系統範圍的配額，不管使用者配額大於或小於系統範圍的配額。

預設值：停用，沒有限制

檔案大小

指定可同步的最大檔案大小。

如果已指定配額，則有效的最大檔案大小限制會與配額相同。

取消選取核取方塊，以停用「最大檔案大小限制」規則。如果已停用規則，則該值會報告為「沒有限制」。

選取核取方塊以啟用「最大檔案大小限制」規則，然後再指定允許的最大檔案大小 (以 MB 為單位)。

如果已指定配額，則預設的最大檔案大小限制會與配額相同。

考慮系統上的下列需求，以判斷您環境內 iFolder 適當的檔案大小限制：

  • 預期用法

  • 最大的檔案修改的頻率

  • 應用程式使用最大的檔案實際將變更儲存至檔案 (整個檔案或差異) 的方法

  • 檔案由每個成員同步的頻率

  • 共享 iFolder 的使用者數目

  • 使用者會存取本地網路上的 iFolder 或存取 WAN 或網際網路連接的 iFolder。

  • 平均和尖峰時期的可用頻寬

即使您為檔案大小限制設定了一個非常大的值，並且未限制檔案大小的配額，則實際的限制會受到使用者電腦上的檔案系統所控制。例如，FAT32 卷冊擁有 4 GB 減去 1 Byte 的最大檔案大小。

預設值：停用，沒有限制

排除的檔案

指定是否要限制由排除過濾器同步的檔案類型。

輸入副檔名，然後按一下「新增」將其加入清單。

若要排除檔案類型不進行同步化，請選取限制檔案類型清單中檔案類型旁的核取方塊，然後按一下「拒絕」。

若要允許檔案類型以進行同步化，請選取限制檔案類型清單中檔案類型旁的核取方塊，然後按一下「允許」。

若要將某個檔案類型從限制檔案類型清單中刪除，請選取該檔案類型旁的核取方塊，然後按一下「刪除」。

同步化

指定最小同步間隔 (以分鐘為單位)。

如果已啟用此選項，請為每個使用者帳戶指定同步 iFolder 資料的最小間隔。較大的值會有較大的限制。

如果停用此選項，則預設值是 5 分鐘。

加密

指定 iFolder 系統的加密規則。系統範圍設定會取代使用者規則。

之中
選取「啟用」以啟用 ifolder 系統的加密功能。如此可允許使用者為他 (或她) 的 iFolder 設定加密規則。

取消選取「啟用」以停用所有使用者的加密功能。

強制執行
選取「強制執行」以啟用所有使用者的加密功能。當加密規則設定為「強制執行」時，使用者便無法變更其 iFolder 的加密設定。

共享

 之中
根據預設，已啟用 iFolder 共享。取消選取「啟用」以停用 iFolder 系統的共享。套用此規則後，此 iFolder 系統的使用者無法與其他人共享其 iFolder。然而，您可以在使用者層級或 LDAPGroup 層級變更規則設定。

強制
選取「強制執行」以強制執行您為整個系統設定的共享規則。您可以強制執行「啟用共享」及「停用共享」。當您強制執行「停用共享」時，iFolder 與使用者層級共享的規則設定會自動停用，且您將無法變更設定。然而，您可以將規則設為「撤銷」。

    重要：除非您將「共享」選項設為「開啟」，否則無法啟用此選項。


撤銷
選取「撤銷」以移除 iFolder 系統下所有 iFolder 的共享成員。

次要管理員

「次要管理員」標籤可讓您新增、編輯或刪除次要管理員詳細資料。

新增次要管理員：

  1. 按一下「次要管理員」索引標籤，然後按一下「新增」，即可顯示 iFolder 使用者清單。
  2. 選取您要指定為次要管理員的使用者，然後按一下「下一步」。
  3. 若要將群組指定給次要管理員，請從「選取群組」清單中選取想要的選項。若選取的群組已設定彙總磁碟配額限制，則「設定整個群組的彙總磁碟配額限制限制」欄位會填入該值。否則，該欄位會維持空白。
  4. 為次要管理員設定規則權限。

    下表列出您可以為次要管理員設定的規則權限。

    參數

    描述

    每位使用者的 iFolder 規則

     

    指定每個使用者能擁有的 iFolder 數量上限。套用此規則後，每個使用者將只能擁有固定數目的 iFolder。超出使用上限的使用者將會收到違反規則相關的錯誤訊息。如果限制為零，使用者無法建立任何 iFolder。

    此規則設定不會影響使用者已擁有的 iFolder 數目。如果使用者擁有的 iFolder 數量已超過您設定的限制， 使用者仍可擁有這些 iFolder。

    根據預設，會選取「每位使用者的 iFolder」規則的「允許」核取方塊。這表示次要管理員有權為指定群組的使用者設定「每位使用者的 iFolder」規則。若不想讓次要管理員擁有此權限，您必須取消選取「允許」核取方塊。

    磁碟配額 規則

    指定使用者可以使用的最大空間。

    根據預設，會選取磁碟配額規則的「允許」核取方塊。這表示次要管理員有權為指定群組的使用者設定磁碟配額規則。若不想讓次要管理員擁有此權限，您必須取消選取「允許」核取方塊。

    檔案大小 規則

    指定可同步的最大檔案大小。

    根據預設，會選取檔案大小規則的「允許」核取方塊。這表示次要管理員有權為指定群組的使用者設定檔案大小規則。若不想讓次要管理員擁有此權限，您必須清除「允許」核取方塊。

    同步間隔規則

    指定最小同步間隔 (以分鐘為單位)。

    根據預設，會選取同步間隔規則的「允許」核取方塊。這表示次要管理員有權為指定群組的使用者設定同步間隔規則。若不想讓次要管理員擁有此權限，您必須取消選取「允許」核取方塊。

    排除的檔案 清單規則

    指定不同步的檔案類型。

    根據預設，會選取排除的檔案清單規則的「允許」核取方塊。這表示次要管理員有權為指定群組的使用者設定排除的檔案清單規則。若不想讓次要管理員擁有此權限，您必須取消選取「允許」核取方塊。

    共享

    指定 iFolder 是否可在使用者之間共享。

    根據預設，會選取共享規則的「允許修改共享規則」核取方塊。這表示次要管理員有權為指定群組的使用者修改共享規則。若不想讓次要管理員擁有此權限，您必須清除「允許」核取方塊。

    加密規則

    指定 iFolder 系統的加密規則。

    根據預設，會選取加密規則的「允許修改加密規則」核取方塊。這表示次要管理員有權為指定群組的使用者修改加密規則。若不想讓次要管理員擁有此權限，您必須取消選取「允許」核取方塊。

    提供權限

    指定次要管理員可用的提供權限。

    根據預設，會選取「允許使用者提供」核取方塊。這表示次要管理員可將指定群組的使用者提供給 iFolder 多伺服器設定中的任一部現有伺服器。若不想讓次要管理員擁有此權限，請取消選取「允許使用者提供」核取方塊。

    透過使用「允許啟用/停用使用者」核取方塊，您可以將啟用或停用指定群組之使用者的權限指定給次要管理員。根據預設，會選取此核取方塊。若不想讓次要管理員擁有此權限，請取消選取「允許啟用/停用使用者」核取方塊。

    iFolder 上的權限

    指定次要管理員對於指定群組之使用者所擁有的 ifolder 的權限。

    若要允許次要管理員擁有殘留的 iFolder，請確定已選取「允許殘留之 iFolder 的擁有權」核取方塊。根據預設，會選取此核取方塊。若不想讓次要管理員擁有此權限，請清除該核取方塊。

    透過使用「允許啟用/停用 iFolder」核取方塊，您可以將啟用或停用指定群組之使用者擁有的 iFolder 的權限指定給次要管理員。根據預設，會選取此核取方塊。若不想讓次要管理員擁有此權限，請清除「允許啟用/停用 iFolder」核取方塊。

    透過使用「允許修改共享之 iFolder 成員的權限」核取方塊，您可以將修改共享之 iFolder 成員的權限指定給次要管理員。根據預設，會選取此核取方塊。若不想讓次要管理員擁有此權限，請清除「允許修改共享之 iFolder 成員的權限」核取方塊。

    使用「允許刪除 iFolder 的權限」核取方塊，您可以指定次要管理員刪除 iFolders 的權限。根據預設，會選取此核取方塊。若不想讓次要管理員擁有此權限，請清除「允許刪除 iFolder 的權限」核取方塊。

  2. 成功將群組指定給次要管理員之後，請按一下「確定」以返回「系統」頁面，或按一下「重複」以指定更多群組給次要管理員。 

編輯次要管理員詳細資料：

  1. 按一下「次要」索引標籤以顯示次要管理員詳細資料。

  2. 選取次要管理員並按一下「編輯」以顯示由該次要管理員所監控的群組清單。

  3. 選取群組並按一下「編輯」以顯示次要管理員對於該群組所擁有的權限清單。編輯群組權限，並按一下「儲存」以儲存您所做的變更。
     

刪除次要管理員：

  1. 按一下「次要」索引標籤以顯示次要管理員詳細資料。
  2. 選取次要管理員並按一下「刪除」以顯示由該次要管理員所監控的群組清單。
  3. 選取所有群組並按一下「刪除」。刪除次要管理員擁有的所有群組也會刪除次要管理員。
