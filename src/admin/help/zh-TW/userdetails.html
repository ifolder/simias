使用者詳細資料

使用此介面檢視使用者詳細資訊，以及管理使用者規則。

注意：iFolder 使用者一詞代表的包括個別使用者和 LdapGroup。

規則

使用者詳細資料

「使用者詳細資訊」頁將會顯示下列選定使用者之 iFolder 帳戶的使用者詳細資訊。

參數

描述

使用者名稱

為使用者帳戶指定的使用者名稱，例如：jsmith 或 john.smith@example.com。

全名

使用者帳戶的名字和姓氏。

成員類型

顯示使用者目前用來登入的成員類型。如果使用者為個別使用者，介面會同時列出使用者群組選項。如果使用者為 LDAPGroup 的成員，介面會在「群組成員」選項下列出所有 LDAPGroup 的成員。

LDAP 網路位置

在 iFolder 內用於提供使用者之 LDAP 網路樹的網路位置。

上次登入

使用者上一次登入 iFolder 系統的時間。

群組磁碟配額 代表群組的彙總磁碟配額。

注意：次要管理員無法修改群組磁碟配額。

已用的磁碟空間 代表群組所使用的彙總磁碟空間。

使用者群組 (僅適用個別使用者)

列出所有選定使用者所屬的群組。

群組成員 (僅適用 LDAPGroup)

列出所有屬於選定 LDAPGroup 的成員。

iFolder

「使用者詳細資料」頁會顯示使用者擁有或共用的 iFolder。按一下「全部」索引標籤，以列出所有擁有和共用的 iFolder。若要檢視使用者擁有的 iFolder，請按一下「擁有」索引標籤。「共用」索引標籤會列出目前使用者帳戶的所有共用的 iFolder。

啟用或停用 iFolder

  1. 找出您要管理的 iFolder，然後選取 iFolder 旁的核取方塊。

  2. 按一下「啟用」以啟用 iFolder。

    如此將可讓使用者登入及同步化 iFolder。

  3. 按一下「停用」以停用 iFolder。

    當您在進行這項變更時，如果使用者已登入，則使用者的會期將會持續到使用者登出為止。規則會在下一次使用者嘗試登入帳戶時生效。若要鎖定使變更立即生效，您必須重新啟動 iFolder 伺服器的 Apache 服務，如此將會中斷所有使用中會期，包括使用者的會期。

刪除 iFolder

  1. 找出您要刪除的 iFolder，然後選取 iFolder 旁的核取方塊。

  2. 按一下「刪除」。

建立 iFolder

  1. 在「使用者詳細資料」頁上，按一下「新增」。

  2. 在開啟的新頁面上，遵循指示以建立新的 iFolder。

規則

使用此介面管理指定之使用者帳戶的使用者規則。

注意： 設定於 LDAPGroup 的規則適用其成員使用者。

修改使用者規則

下表會列出您可以為任何指定 iFolder 使用者帳戶管理的規則。修改下列一或多個值，然後按一下「儲存」套用新設定：

參數

描述

帳戶

指定使用者目前是否可以登入以同步化 iFolder。

選取核取方塊以停用帳戶的登入。

取消選取核取方塊以啟用帳戶的登入。

當您在進行這項變更時，如果使用者已登入，則使用者的會期將會持續到使用者登出為止。規則會在下一次使用者嘗試登入帳戶時生效。若要鎖定使變更立即生效，您必須重新啟動 iFolder 伺服器的 Apache 服務，如此將會中斷所有使用中會期，包括使用者的會期。

預設值：已啟用

每個使用者可擁有的 iFolder 數目

指定每個使用者可擁有的 iFolder 數目上限。套用此規則後，使用者將只能擁有固定數目的 iFolder。超出使用上限的使用者將會收到違反規定相關的錯誤訊息。如果限制為零，使用者無法建立任何 iFolder。

選取「限制」旁的核取方塊以啟用每個使用者的 iFolder 限制，然後在欄位中指定數量。

規則設定不影響使用者已擁有的 iFolder 數目。如果使用者擁有的 iFolder 數量超過設定的限制，使用者仍可擁有其 iFolder。

使用者層級規則會覆寫 LDAPGroup 層級與系統層級規則。

預設值：停用，未設定值

磁碟配額

指定使用者的 iFolder 資料可用的最大總空間，包括使用者擁有的所有 iFolder。您可以使用「系統範圍」規則加入配額。使用者規則會取代系統範圍規則。

限制
指定使用者之 iFolder 資料允許使用的最大總空間 (以 MB 為單位)，包括選定使用者帳戶之使用者擁有的所有 iFolder。

如果沒有個別的使用者配額，請取消選取此選項，或者接受選定使用者帳戶的 LDAPGroup 層級配額或系統範圍配額。
選取此選項以強制執行使用者配額，然後再為選定使用者帳戶指定總配額空間 (以 MB 為單位)。

已使用
指定目前伺服器上供所有選定使用者擁有之 iFolder 使用的總空間。

可用的
伺服器上可用的可使用的空間。

有效
伺服器上配置的有效空間。

如果 iFolder 資料消耗的總空間接近有效配額 (系統、LDAPGroup、使用者或 iFolder)，使用者應該停止同步檔案，直到接下來的一或多個工作使得空間足以在檔案所在的 iFolder 內安全地進行使用者檔案同步為止：

  • 系統範圍的配額、LDAPGroup 的配額、iFolder 擁有者的使用者配額，以及 iFolder 配額會視需要進行修改。

  • 檔案會從使用者擁有的任何 iFolder 移動至另一個位置，而這個位置不再受到有效配額的影響，或者會刪除檔案以清除空間。

重要：如果您啟用某個使用者空間限制，而這個限制小於使用者目前的 iFolder 資料的總空間，則使用者的資料會停止同步，直到資料減少至限制範圍內，或將配額值增加為大於使用者所消耗的總空間為止。

預設值：停用且沒有設定配額

檔案大小

 限制
指定 iFolder 使用者可以使用的最大總檔案大小 (以 MB 為單位)，包含選定使用者帳戶之使用者擁有的所有 iFolder。

有效
為使用者配置的有效檔案大小。

    重要：使用者無法成功地同步檔案大小將會超過配額的檔案。如果他們嘗試如此做，則只有部份檔案會進行同步，如此可能會導致資料的毀損。

預設值：停用且沒有限制

排除的檔案

為選定使用者的帳戶指定要納入同步或從同步排除的檔案類型清單。

檔案管理員名為 thumbs.db 和 .DS_Store 的檔案永遠不會進行同步。您不需要保留這些檔案，將它們同步會導致重複的檔案衝突錯誤。如果您未為此使用者設定任何個別的限制，此欄位會將 thumbs.db 和 .DS_Store 報告為系統範圍案類型限制的一部份。為使用者設定個別的檔案類型限制後，便會改以顯示使用者的設定。即使未顯示 thumbs.db 和 .DS_Store 限制，也會套用這些限制，您無法覆寫這些限制。

若要將副檔名加入包含或排除過濾器，請輸入副檔名 (例如 *.mpg)，然後按一下「新增」套用過濾器。

若要排除檔案類型不進行同步化，請選取限制檔案類型清單中檔案類型旁的核取方塊，然後按一下「拒絕」。

若要允許檔案類型以進行同步化，請選取限制檔案類型清單中檔案類型旁的核取方塊，然後按一下「允許」。

若要將某個檔案類型從限制檔案類型清單中刪除，請選取該檔案類型旁的核取方塊，然後按一下「刪除」。

預設值：DS_Store 和 Thumbs.db

同步化

指定最小同步間隔 (以秒為單位)。您可以將同步間隔設定為包含介於 5 和 999 之間的值。如果未設定值，則會將值設為 5。

取消選取核取方塊不設定同步間隔，或接受使用者帳戶的系統範圍設定。如果設定了系統範圍或使用者規則的值，則報告的值會是「沒有限制」。

間隔
選取核取方塊以啟用最小同步間隔，然後再指定最小間隔 (以分鐘為單位)。例如，實際值為 10 分鐘。

有效
指定目前的同步間隔。例如，若使用者設定了小於 (更頻繁) 系統最小值的同步間隔，則會套用系統設定值。

有效的最小同步間隔一定會是下列設定中最長的值：

  • 系統規則 (預設為零)，除非已設定 LDAPGroup 規則或使用者規則。如果設定使用者規則，則使用者規則會覆寫系統規則和 LDAPGroup 規則，而不論使用者間隔的值較之是較短還是較長。

  • 本地機器規則，或與伺服器同步化之用戶端機器的設定。

  • iFolder (集合) 規則。

預設值：停用，或系統範圍的規則

加密

為選定的使用者帳戶指定加密規則。配額可以藉由系統規則 (全系統) 增加。全系統規則會取代使用者規則。

之中
選取「啟用」以啟用選定使用者的加密功能。如此可允許使用者為他 (或她) 的 iFolder 設定加密規則。

啟用加密功能時無法將其關閉。

強制執行
選取「強制執行」以啟用選定使用者的加密功能。當其設定為「強制執行」時，使用者便無法變更他 (或她) 的 iFolder 加密設定。

重要：只有在系統層級的加密規則設定為「啟用」時，這個選項才會啟用。

共享

 之中
根據預設，已啟用 iFolder 共享。取消選取「啟用」，然後按一下「儲存」以停用選定使用者的共享。套用此規則後，選定的使用者將無法與他人共享其 iFolder。然而，您仍可以在 iFolder 層級變更規則。

強制
選取「強制執行」以強制執行為選定使用者設定的規則。套用此規則後，使用者將無法共享其 iFolder。

撤銷
選取「撤銷」以移除選定使用者所屬所有 iFolder 的共享成員。
