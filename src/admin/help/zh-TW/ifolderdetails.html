描述 (可編輯)

iFolder 的簡短描述。您可以編輯敘述並按一下「儲存」以儲存變更。

擁有者

選定 iFolder 之擁有者的使用者名稱。擁有者可以是個別使用者或 LDAPGroup。如果 LDAPGroup 為 iFolder 的擁有者，則所有此 LDAPGroup 的成員都擁有相同的存取權限。對於殘留下來的 iFolder，「iFolder 管理員」使用者會成為擁有者，直到 iFolder 可以重新指派或刪除為止。

按一下擁有者的使用者名稱連結，以檢視他 (或她) 的帳戶詳細資料。

iFolder 擁有者具有 iFolder 的「完全控制」權限。擁有者可管理成員資格和使用者的存取權限，並且可以移除任何成員的「完全控制」權限。

路徑

iFolder 實際的位置，以及其位於伺服器上的資料。例如：

/var/opt/novell/ifolder3/simias/SimiasFiles/e84fdc6e-3d51-49df-ae3f-8c9213c76994/<iFolder_Name>

此例中，e84fdc6e-3d51-49df-ae3f-8c9213c76994 是 iFolder 共用的獨特 ID。

成員

您可以檢視成員的名稱、類型和指定給他們的存取權限。您可以加入或刪除擁有者、指定擁有權，以及設定選定成員的存取權限。

加入成員

  1. 在「iFolder 詳細資料」頁上，按一下「新增」。

  2. 搜尋您要將其加入為成員的使用者或 LDAPGroup，選取使用者名稱旁的核取方塊，然後按一下「新增」。

    依預設值，使用者或 LDAPGroup 將僅能獲得 iFolder 上的「唯讀」存取權限。

  3. (選擇性) 選取「使用者」核取方塊，然後選取「權限」並將「存取權限」指定為「管理員」、「讀/寫」或「唯讀」，然後按一下「設定」。

    等待頁面重新整理。使用者的圖示應該會反映新的存取權限。

瞭解 iFolder 存取權限

iFolder 用戶端支援 iFolder 的擁有者，並支援其成員的三個存取層級：

注意： LDAPGroup 的成員會繼承為該 LDAPGroup 設定的存取權限。

擁有者
僅有一個使用者或 LDAPGroup 可做為 iFolder 的擁有者。通常這是建立此 iFolder 的 LDAPGroup 之使用者或任何成員。如果擁有者為 LDAPGroup，則所有此 LDAPGroup 的成員都具有此 iFolder 的擁有權。擁有者 LDAPGroup 的擁有者使用者或成員可以使用 iFolder 用戶端將擁有權轉讓給 iFolder 中其他成員。「iFolder 管理員」使用者可以將擁有權轉讓給 iFolder 伺服器上的任何使用者或 LDAPGroup。

iFolder 的擁有者具有「完全控制」權限。此使用者或 LDAPGroup 具有 iFolder 的讀/寫存取權，可管理成員資格及成員使用者及 LDAPGroup 的存取權限，還可以移除任何成員的「完全控制」權限。使用企業伺服器時，由擁有者的 iFolder 計數使用的磁碟空間是依照企業伺服器上擁有者的使用者磁碟配額決定的。

如果將使用者或 LDAPGroup 從 iFolder 系統中刪除，此使用者或 LDAPGroup 所擁有的 iFolder 將成為殘餘的 iFolder。殘留的 iFolder 會暫時將「iFolder 管理員」使用者指定為擁有者。其他使用者和 LDAPGroup 成員的成員資格和同步會持續進行，同時「iFolder 管理員」使用者會判斷應該將殘餘的 iFolder 刪除，或為其指派新的擁有者。

完全控制
共用 iFolder 的成員，具有「完全控制」存取權限。具有存取 iFolder「完全控制」權限的使用者或 LDAPGroup 可管理除了擁有者之外，所有使用者的成員資格及存取權限。

讀/寫
共用 iFolder 的成員，具有 iFolder 中目錄及檔案的「讀╱寫」存取權限。

唯讀
共用 iFolder 的成員，具有 iFolder 中目錄及檔案的「唯讀」存取權限。這個成員可以將 iFolder 檔案複製到另一個位置，並在 iFolder 外部修改該檔案。

設定成員的 iFolder 存取權限

  1. 在「iFolder 詳細資料」頁上，選取您要管理其存取權限之成員旁的核取方塊。

  2. 選取「權限」下拉式功能表，然後選取「管理員」、「讀/寫」或「唯讀」權限。

    等待頁面重新整理。使用者的圖示應該會反映新的存取權限。

移除成員

  1. 在「iFolder 詳細資料」頁上，選取成員使用者名稱旁的核取方塊。
  2. 按一下「刪除」。

使用者的本地資料副本將會保留在使用者的電腦上，但是使用者無法再存取 iFolder 伺服器的伺服器副本。

變更擁有者

當您變更 iFolder 的擁有者時，現有的擁有者會變成 iFolder 的成員，並為其指定「讀/寫」權限。對於殘留下來的 iFolder，「ifolder 管理員」會成為擁有者，直到為 iFolder 重新指定新的擁有者或將其刪除為止。如果新的擁有者目前不是 iFolder 的成員，則會傳送訂閱通知到使用者的 iFolder 帳戶。

  1. 在「ifolder 詳細資料」頁上，搜尋您要將其指定為 ifolder 新擁有者的使用者或 LDAPGroup。

  2. 選取使用者名稱旁的核取方塊，然後按一下「擁有者」。

規則

下表列出您可以為任何指定 iFolder 管理的規則。修改下列一或多個值，然後按一下「儲存」套用新設定：

停用同步

選取此選項關閉同步，通常是暫時性的。

取消選取此選項，啟用 iFolder 內的資料同步。

磁碟配額

選取「限制」核取方塊，然後為選定的 iFolder 指定最大的大小 (以 MB 為單位)。

如果您啟用了系統範圍的 iFolder 配額或 LDAPGroup 層級配額，則當使用者配額低於或高於 LDAPGroup 配額或系統配額時，使用者帳戶配額將會覆寫系統範圍的配額或 LDAPGroup 層級配額。

預設值：停用，無限制

檔案大小

 限制
指定 iFolder 使用者可以使用的最大總檔案大小 (以 MB 為單位)，包含選定使用者帳戶之使用者擁有的所有 iFolder。

有效
為使用者配置的有效檔案大小。

    重要：使用者無法成功地同步檔案大小將會超過配額的檔案。如果他們嘗試如此做，則只有部份檔案會進行同步，如此可能會導致資料的毀損。

排除的檔案

為選定 iFolder 指定要納入同步或從同步排除的檔案類型清單。

檔案管理員名為 thumbs.db 和 .DS_Store 的檔案永遠不會進行同步。

若要將副檔名加入排除過濾器，請輸入副檔名 (例如 *.mpg)，然後按一下「新增」套用過濾器。

若要排除檔案類型不進行同步化，請選取限制檔案類型清單中檔案類型旁的核取方塊，然後按一下「拒絕」。

若要允許檔案類型以進行同步化，請選取限制檔案類型清單中檔案類型旁的核取方塊，然後按一下「允許」。

若要將某個檔案類型從限制檔案類型清單中刪除，請選取該檔案類型旁的核取方塊，然後按一下「刪除」。

預設值：停用、允許所有檔案類型或系統範圍的設定

同步間隔

選取「同步間隔」核取方塊以啟用選定 iFolder 的最小間隔設定，然後再以分鐘為單位指定最小值，使用者在這個時間內可以設定他們的 iFolder 用戶端。

若要停用設定，請取消選取「同步間隔」核取方塊。如果選項已停用，則報告的值會為「沒有限制」。

如果已啟用這個選項，則最小同步間隔會以分鐘為單位，指定使用者的用戶端可以檢查伺服器和本地 iFolder 之 iFolder 資料的最小間隔，以識別需要下載或上傳的檔案。

有效的最小同步間隔一定會是下列設定的最大值：

  • 系統規則 (預設為 5 分鐘)，除非已設定使用者規則或 LDAPGroup 規則。如果設定 LDAPGroup 規則，則 LDAPGroup 規則會覆寫系統規則。如果設定使用者規則，則使用者規則會覆寫 LDAPGroup 規則和系統規則，而不論使用者規則的值較之是大還是小。

  • 本地機器規則，或與伺服器同步化之用戶端機器的設定。

  • iFolder (集合) 規則。

預設值：停用，用戶端設定限制

共享

 之中
根據預設，已啟用 iFolder 共享。取消選取「啟用」以停用選定 iFolder 的共享。套用此規則後，選定的 iFolder 將不能讓管理員或擁有者共享。

撤銷
選取「撤銷」以從選定 iFolder 的共享成員清單上移除所有成員。

    重要：若您在系統或使用者層級啟用並強制「共享」選項，則兩個選項都將停用。
