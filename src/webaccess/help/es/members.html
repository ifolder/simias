Gestión de los miembros de una carpeta iFolder

Esta interfaz permite añadir o eliminar miembros compartidos, asignar derechos a los miembros y cambiar la propiedad de la carpeta iFolder seleccionada.

En esta ayuda se tratan los temas siguientes:

Gestión de usuarios de una carpeta iFolder
Explicación de los derechos de acceso de iFolder

Gestión de usuarios de una carpeta iFolder

En una carpeta iFolder es posible realizar las siguientes tareas:

Adición de un nuevo usuario compartido

Haga clic en Añadir para añadir nuevos miembros compartidos a la carpeta iFolder seleccionada. Se puede añadir un usuario de iFolder individual o un grupo LDAP como miembro compartido. La nueva página que aparece permite añadir usuarios a la lista de usuarios compartidos.

Asignación de derechos a un usuario compartido

En la página Miembros, seleccione los miembros compartidos y haga clic en los derechos que desee asignar a éstos.

Explicación de los derechos de acceso de iFolder

El Cliente iFolder es compatible con el propietario y tres niveles de acceso para miembros en una carpeta iFolder:

IMPORTANTE: todos los miembros de un grupo LDAP heredan el conjunto de derechos de acceso de dicho grupo.

Sólo lectura
Miembro de una carpeta iFolder compartida con derecho de acceso de sólo lectura a los directorios y archivos de dicha carpeta iFolder. Este tipo de miembro puede copiar un archivo de la carpeta iFolder a otra ubicación y modificarlo fuera de dicha carpeta.

Lectura y escritura
Miembro de una carpeta iFolder compartida con derecho de acceso de lectura/escritura a los directorios y archivos de dicha carpeta iFolder.

Control total
Miembro de una carpeta iFolder compartida con derecho de acceso de control total. El usuario con derecho de control total tiene acceso de lectura/escritura a la carpeta iFolder y gestiona la pertenencia y los derechos de acceso de todos los usuarios exceptuando al propietario. Sin embargo, sólo el propietario tiene el derecho de suprimir una carpeta iFolder.

Propietario
Sólo un usuario puede ser el propietario de una carpeta iFolder. Por lo general, se trata del usuario que ha creado la carpeta iFolder, y sólo éste podrá suprimir la carpeta iFolder. El usuario propietario puede utilizar el Cliente iFolder para transferir la propiedad a otro miembro de la carpeta iFolder. El usuario Admin de iFolder puede transferir la propiedad a cualquier usuario del servidor iFolder.

El propietario de una carpeta iFolder tiene el derecho de control total. Este usuario tiene acceso de lectura/escritura a la carpeta iFolder, gestiona la pertenencia y los derechos de acceso de los usuarios miembro y puede quitar el derecho de control total a cualquier miembro.

Si se suprime un usuario como usuario del sistema de iFolder, las carpetas iFolder propiedad de dicho usuario quedarán huérfanas. Las carpetas iFolder huérfanas se asignan temporalmente al usuario Admin de iFolder, que pasará a ser el propietario. La pertenencia y sincronización siguen aplicándose para los demás usuarios, mientras que el usuario Admin de iFolder determina si la carpeta iFolder huérfana en cuestión debe suprimirse o asignarse a un nuevo propietario.
