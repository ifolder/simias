管理 iFolder 的用户

在 iFolder 中，您可以执行以下任务：

添加新共享用户

单击“添加”可在选定 iFolder 中添加新的共享成员。您可将单个 iFolder 用户或 LDAP 组添加为共享成员。新页打开，您可在其中将用户添加到共享用户列表。

为共享用户指派权限

在“成员”页面上，选择共享成员，然后单击要指派给他们的权限。

了解 iFolder 访问权限

iFolder 客户程序支持拥有者和三个 iFolder 成员访问级别：

重要说明：LDAP 组的所有成员都将继承为该 LDAP 组设置的访问权限。

只读
共享 iFolder 的成员，对 iFolder 中的目录和文件具有“只读”访问权限。该成员可将 iFolder 文件复制到另一个位置并在 iFolder 外修改它。

读写
共享 iFolder 的成员，对 iFolder 中的目录和文件具有“读/写”访问权限。

完全控制
共享 iFolder 的成员，具有“完全控制”访问权限。具有“完全控制”权限的用户可以对 iFolder 进行读/写访问，并管理除拥有者之外所有用户的成员资格和访问权限。但是，只有拥有者具有删除 iFolder 的权限。

拥有者
只有一个用户担当 iFolder 的拥有者。这通常是创建 iFolder 的用户并且只有他或她可以删除该 iFolder。拥有者用户可以使用 iFolder 客户程序将所有权转让给该 iFolder 的另一个成员。iFolder Admin 用户可将所有权转让给 iFolder 服务器上的任何用户。

iFolder 的拥有者具有“完全控制”权限。此用户可以对 iFolder 进行读/写访问，管理成员用户的成员身份和访问权限，并且可以去除任何成员的“完全控制”权限。

如果将用户作为 iFolder 系统的用户删除，则会孤立该用户的 iFolder。孤立的 iFolder 的所有权被临时指派给 iFolder Admin 用户。当 iFolder Admin 用户确定是否删除孤立的 iFolder 或是否将它指派给新的拥有者时，其他用户的成员资格和同步状态保持不变。
